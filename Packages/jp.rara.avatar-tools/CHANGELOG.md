# Changelog

## [1.3.0] - 2026-07-18
### Added
- Quest変換の対応元シェーダーを拡張しました。これまでの lilToon / NonToon に加えて、Poiyomi Toon(ロック=Lock In 済みを含む)・Unity 標準(Standard / Standard (Specular setup))・Unity Unlit 系(Unlit/Color・Texture・Transparent・Transparent Cutout)・VRM MToon(VRM0.x「VRM/MToon」/ VRM1.0「VRM10/MToon10」)を、それぞれ専用の変換経路で Toon Standard / Toon Lit へ変換します。従来これらは「メインテクスチャのみの汎用近似」に落ちて警告が出ていましたが、系統ごとにメインテクスチャ(×色)・ノーマルマップ・カリング・エミッション・マットキャップ等を適切に読み取って対応付けるようになりました(汎用近似の警告は、本当に未対応のシェーダーだけに出ます)。
- Poiyomi Toon: アルベド(メイン × カラー、Main Color Adjust の色相/彩度/明度/ガンマ補正)をベイクし、ノーマルマップ・エミッション・カリングを対応付けます。ロック済みマテリアルはタグから元テクスチャを復元して読み取ります。影ランプ・マットキャップ・リム等の一部の質感はシェーダー内部情報が参照できないため既定値へ縮退し、その旨をレポートします。
- Unity 標準 / Unlit: メインテクスチャ × カラーをベイクし、Standard はノーマルマップを引き継いで既定の2バンド影ランプ、Unlit は陰影を持たないためフラット(均一)ランプで変換します。メタリック・スムースネスは Toon Standard に相当機能が無いため反映されません。金属パーツはマテリアル別指定の「MatCap Lit(金属向け)」に切り替えると映り込みで近似できる旨を案内します。
- VRM MToon: メインテクスチャ × カラーをベイクし、ノーマルマップ・カリングを対応付けます。加算スフィア(_SphereAdd / MToon10 の _MatcapTexture)は Toon Standard の加算マットキャップへ引き継ぎ、トゥーン影(Shade 色・Toony・Shift)は Toon Standard に色付き影枠が無いため既定ランプで近似します。アウトラインは Quest 非対応のため失われます。半透明・カットアウトは _BlendMode で正しく判定します(RenderType タグが Opaque 固定で信頼できないため)。
- 上記いずれの新系統も、半透明(アルファブレンド)マテリアルは既存の透過処理設定(再現=乗算/加算 / 非表示 / 不透明化)に従い、マテリアル別の変換方法指定(Toon Standard / Toon Lit / MatCap Lit / パーティクル(加算・乗算) / 非表示 / 変換しない)も利用できます。エミッションは既存の汎用ベイク経路でメインテクスチャへ焼き込む近似です(この波ではシェーダー固有の追加GPUベイクは行いません)。

## [1.2.0] - 2026-07-18
### Added
- PC/Quest間で揺れ物を掴んだときの同期ズレを防ぐNetwork ID自動割り当てを追加しました。Questコンバーターの「_Quest」複製とPC軽量化の「_Opt」複製に、元アバターを基準にした一貫したNetwork ID(PhysBoneなどの揺れ物)を書き込むため、同じ揺れ物が両プラットフォームで同じIDになります。これにより、他ユーザーから見た掴み・ポーズ・ストレッチの同期がPC版とQuest版でズレる問題を防げます。既定でオンで、各ツールの詳細設定(Questコンバーター)/実行セクション(PC軽量化)の「Network IDを割り当てる(PC/Quest間の揺れ物の掴み同期)」でオン/オフできます。元アバターに既にNetwork IDが割り当てられている場合はそのIDを引き継ぎます。

## [1.1.0] - 2026-07-18
### Added
- lilToonのマットキャップ(金属・宝石等の映り込み)をToon Standardへ自動で引き継ぐようにしました(既定オン。変換設定詳細の「マットキャップを引き継ぐ」で切り替え可能)。マットキャップテクスチャ・合成強度・合成モード(乗算のみ乗算・それ以外は加算近似)・合成マスクを対応付けます。従来はマットキャップは変換されず警告のみでしたが、Toon Standardのネイティブマットキャップで映り込みが維持されます(独自UV・視差・ディテール法線・その2(2nd)マットキャップは非対応)。アトラス統合では、異なるマットキャップが1マテリアルに混ざらないよう同一マットキャップのマテリアルだけを統合し、マットキャップマスク付きは正しさを優先して統合対象から外します(PC最適化のアトラスでも異なるマットキャップは統合しません)。
- マテリアル別の変換方法に「MatCap Lit(金属向け)」を追加しました。金属パーツ向けの VRChat/Mobile/MatCap Lit へ変換する選択肢で、メインテクスチャに映り込み(マットキャップ)を乗算する軽量シェーダーです。元マテリアルのマットキャップ画像があれば引き継ぎ、無ければ白(素のライティング相当)で変換して割り当てを案内します。不透明のみ・乗算合成のマットキャップで、アトラス統合の対象外のためスロットを1つ使います。
### Fixed
- 変換でポリゴン数が増える問題を完全に解消しました。マテリアルスロット数がサブメッシュ数を超えるメッシュ(多重描画スロット)で、1.0.8 は「見た目に効く上描き」を専用サブメッシュへ実体化していたため三角形が水増しされていました(Kei髪の実測例: 68,674 → 78,948)。1.0.9 は三角形の複製を全廃し、上描きスロットを内容で振り分けます。何も描かない非表示(不可視)の上描き(Questで消える疑似影 FakeShadow など)は削除し、基底と同じ材質・同じアトラスセルに重なる冗長な上描きも削除します(いずれもポリゴン増ゼロ)。それ以外の実際に見える上描きは、元アバターと同じネイティブ多重描画のまま保持し(複製しない)、基底サブメッシュのUVを再配置せず上描きと同じUVを共有させることで見た目を維持します。結果、Kei髪は Quest 版=1スロット(疑似影を削除)・PC 版=元と同じ2スロット(髪本体＋疑似影がそのまま描画)となり、どちらも 68,674 のまま増えません。PC最適化・Quest変換の両アトラス経路に適用。
- 上記の変更に伴い、生成するアトラスメッシュ/アトラスアセットの命名規則を更新しました。同じアトラス配置でも 1.0.8 と 1.0.9 では別名のアセットとして出力されるため、シーンに残った 1.0.8 の複製(アップロード直前アバター等)が、内容の異なる同名アセットで上書きされて壊れることはありません(旧複製は旧アセットのまま整合して残ります)。

## [1.0.8] - 2026-07-17
### Fixed
- 変換後にポリゴン数が増える問題を修正。マテリアルスロット数がサブメッシュ数を超えるメッシュ(多重描画スロット)で、旧実装が余剰スロットを一律に実サブメッシュへ複製していたため三角形が水増しされていました(実測例: 68,674 → 78,948)。新実装では、基底と同一マテリアル/同一アトラスセルへ写像される冗長スロットは複製せず統合し(ポリゴン増ゼロ)、見た目に効く真の上描きスロットのみ専用サブメッシュ化して正しいUVを与えます。PC最適化・Quest変換の両アトラス経路に適用。
- 設定が別プロジェクト・別アバターへ漏れる問題を修正。EditorPrefs はマシン全体で共有されるため、旧ツールからの設定引き継ぎで別アバター固有のパスリスト(トグル選択・PhysBone保持リスト等)が新規プロジェクトへ混入し、「トグルパスが見つかりません」警告が大量に出ていました。設定キーをプロジェクト単位で分離し、引き継ぎはアバター非依存の共通設定のみに限定。パス不一致の警告は1件のまとめメッセージに集約しました。
- PCアトラス化で顔・体・髪のテクスチャが過度に縮小される問題を修正(実測例: 顔が1024アトラス内で128pxまで縮小)。品質フロア(顔/体/髪 ≥512px・その他 ≥256px・色のみ 16px)を導入し、フロアでも1枚に収まらない場合はアトラスを複数枚に自動分割して品質を守ります。PC側のアトラスサイズ選択に 4096 を追加(既定は 2048 のまま)。

## [1.0.7] - 2026-07-17
### Added
- Meshia Mesh Simplification（Ram.Type-0 氏 / MIT）との併用ガイドを追加。ポリゴン削減の解説付近に「Meshiaとの併用（推奨）」節を新設しました（Meshiaの概要・VPM導入リンク・役割分担と3つの併用パターン・削減はビルド時反映でエディタ/診断の数値に出ない点・二重削減の注意（目標の置き方）・謝辞）。対象: README・Docs（AvatarStudio）・導入ページ（Website）。
- ⑤ ポリゴン削減パネルに Meshia 検出の案内を追加。選択アバターに Meshia コンポーネント（MeshiaMeshSimplifier / MeshiaCascadingAvatarMeshSimplifier）が付いていると、「Meshia検出: ビルド時にさらに削減されます。本ツールのポリゴン削減と併用する場合は削りすぎに注意（どちらか一方を主にする）」のInfo表示と公式ドキュメントを開くボタンを出します（リフレクション検出・アバター単位でキャッシュ・Meshia未導入時は非表示）。
- 実測レポート（ビルド/Play 時の実測）を追加。メニュー RARA / 実測レポート から開くウィンドウで、_Opt / _Quest 複製を ▶️(Play)、または SDK の Build & Test / Upload でビルドすると、MA / AAO / Meshia / lilToon 適用後・EditorOnly 除去後の最終複製そのものを VRChat SDK と同じ計算で実測し、ポップアップ表示します（PC / Quest 両基準の総合ランク・項目別値・目標達成判定・元アバターとの差。アップロードビルドでは生成アセットバンドルの実ファイルサイズも記録し、Quest/Android は 10MB 判定つき）。計測は上部トグル「ビルド/Play時に実測する」でオフにでき、自動表示だけを止めるトグルも別にあります。既定は _Opt / _Quest 複製のみ対象（全アバター計測トグルあり）。ドキュメントは「エディタ数値は控えめ」の既存注意から本機能へクロスリンクしました。対象: README・Docs（AvatarStudio / QuestConverter）・導入ページ（Website）。
- SkinnedMesh統合にグループ指定モードを追加（統合しない / 顔以外を統合(推奨) / グループ指定で統合 の3モード）。レンダラーをグループ1〜8に振り分け、まとまりごとに1つのメッシュへ統合できます（顔は常に自動保護でグループ割り当て不可、未割り当ては統合しない）。使い方・自動保護の解説をドキュメントへ追記。対象: README・Docs（AvatarStudio / QuestConverter）・導入ページ（Website）。
- 隠れメッシュ削減の検出の仕組み（名前トークンでの候補化・顔まわりの名前/ジオメトリ両面除外・約40%以上を動かす形状の幾何判定）と「その服を後で非表示にすると下の肌が無くなり穴が開く」注意を、パネル説明と同内容でドキュメントへ要約追記。対象: Docs（QuestConverter / AvatarStudio）・README・導入ページ（Website）。
- AAO 連携の解説に「PhysBone のマージは本ツール独自実装で、AAO の Merge PhysBone は使用していない」旨を1行追記。対象: README・Docs・導入ページ（Website）。
