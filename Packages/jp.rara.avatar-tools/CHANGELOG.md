# Changelog

## [1.6.0] - 2026-07-19
### Changed
- ポリゴン削減を Meshia Mesh Simplification(Ram.Type-0 氏 / MIT・NDMF)連携へ移行しました。本ツールが独自に持っていたメッシュ簡略化(1.5系までの「目標へ配分」方式)は廃止し、⑩ポリゴン削減セクションは「ポリゴン削減(Meshia連携)」パネルへ置き換えました。目標ランク(7,500 / 10,000 / 15,000 / 20,000)またはカスタムで目標三角形数を選び、変換時に複製へ Meshia の簡略化コンポーネント(アバター単位の Cascading・自動配分オン)を付与します。実際のポリゴン削減はビルド時(アップロード/Play時=NDMF)に適用されるため、生成直後の複製やエディタ上の三角形数には出ませんが、変換直後の即時実測(1.4.0)には反映されます。元アバターは変更しません。
- Meshia(および Modular Avatar)が未導入のときは、ポリゴン削減パネルに導入案内と導入ページ(VPM)ボタンを表示し、「導入後にこのセクションから設定できます」と案内します。
- 旧バージョンで作成したポリゴン削減の割り当て計画は適用されなくなりました(設定ファイルに残っていても無視されます)。該当する計画が残っている場合は、パネルと変換レポートに「ポリゴン削減は Meshia 連携へ移行しました(旧バージョンの削減計画は適用されません)」と一度だけ通知します(設定項目は互換性のため残置)。診断の「目標達成ガイド」のポリゴン超過アドバイスも、Meshia 連携のセクションを指すようにしました。
- わかりやすさの改善として、ラベル・ツールチップ・メッセージの表記を全体で統一・平易化しました(動作は変わりません)。主な変更: トグル整理の選択肢を「トグル維持 / 常時表示 / 非表示除去(削除)」に統一(「非表示除去」は実際にメッシュを削除することを明示)、半透明の扱いのドロップダウンを日本語表示(「自動で半透明を再現(推奨)/ 非表示にする(最軽量)/ 不透明に変換」)へ、マテリアル変換方法の呼称を全ツールで統一(「自動(推奨)」「加算(半透明)」「乗算(半透明)」など・失われていた各選択肢の説明ツールチップも復活)、プリセット名を「おまかせQuest対応」「フル両対応(PC+Quest)」「PCを最低Poorまで軽量化(Very Poor回避)」等に、Questのアップロード可否は圧縮後10MB・展開後40MBの両上限で決まる旨の明記、ビセーム→「口パク」・NDMF/AAOの用語注記の追加、Very Poor 時の影響(他の人には既定で表示されず揺れものも停止)の明記、揺れもの・スキンメッシュなどの表記ゆれの統一。

## [1.5.1] - 2026-07-19
### Changed
- EditorOnly タグ付き・Quest除外に登録したオブジェクトを、診断だけでなく各ツールの一覧・プレビューと変換からも除外するようにしました。これらはビルドに含まれないため、マテリアル一覧・衣装/トグル整理(削減量チップ含む)・SkinnedMesh統合プレビュー・PhysBone一覧・ポリゴン削減の割り当て・隠面メッシュ候補・テクスチャ一覧などの選択対象から自動で隠れ、統合前後の概算にも数えません(Quest除外の登録パネルだけは全件を表示し、既にEditorOnlyタグが付いているものには「EditorOnly」バッジを付けます)。非表示にした項目があるセクションには「EditorOnly/Quest除外のため n 件を非表示(ビルドに含まれないため)」の注記を1行だけ表示します。
- 変換時に、EditorOnly/Quest除外の配下だけで使われているマテリアルは変換せず(無駄な生成マテリアル・テクスチャを出しません)、レポートに「EditorOnly/Quest除外配下のみで使用のため変換をスキップ: n件」と表示します。複数の場所で使われている(除外配下と通常の両方から参照される)マテリアルは従来どおり変換されます。
### Added
- アップロード手順ガイドに「_Optだけをアクティブ化」(手順①)・「_Questだけをアクティブ化」(手順④)ボタンを追加しました。押すと、アップロードする複製だけをシーンでアクティブにし、関連する既知のアバター(元アバターと、同じ元のもう一方の複製 _Opt/_Quest)だけを非アクティブにします(無関係なアバターや非アバターには触れません。切り替えた各オブジェクトは Undo 登録するので Ctrl+Z で元に戻せます)。SDK の Builder パネルはアクティブなシーン内アバターを一覧するため、誤って別のアバター(PC用シェーダーのままの _Opt や元アバター)を選んでビルドに失敗する事故を防げます。ボタンの隣に現在のアクティブ状態(_Opt / _Quest / 元)を表示します。
- アップロード手順ガイドに、ビルドターゲットとアクティブなアバターの食い違い警告を追加しました。ビルドターゲットが Android/iOS なのに _Quest が非アクティブで元または _Opt がアクティブなときは、赤い警告(SDKパネルでPC用アバターを選んでいると Mobile 非対応シェーダーの警告が大量に出てビルドできない旨)を表示し、手順④の「_Questだけをアクティブ化」を促します。逆に Windows なのに _Opt が非アクティブ・_Quest がアクティブなときは、軽い情報表示を出します。
- アップロード手順ガイドの手順①付近に、SDK の「Multi-Platform Build & Publish」(複数プラットフォーム同時ビルド)を使わないよう常設の注意を追加しました。本ツールは PC 版と Quest 版を別々の複製でアップロードする方式のため、プラットフォームは1つずつ選び、手順①〜④の順に進めます。

## [1.5.0] - 2026-07-18
### Added
- サイズ診断が「圧縮後(ビルド後ダウンロード)10MB」と「展開後(非圧縮メモリ)40MB」を、独立した2つのアップロード上限として表示・判定するようになりました。従来は圧縮後10MBだけを見ていましたが、圧縮後が収まっていても展開後が40MBを超えるとSDKがアップロードをブロックするため、両方を並べて(圧縮後 ◯/10MB・展開後 ◯/40MB)超過している側を明示します。可否フラグも両上限のいずれか超過で「不可」になります。
- テクスチャの「10MB以下まで自動調整」を「上限内まで自動調整(圧縮後10MB・展開後40MB)」に拡張しました。圧縮後・展開後の両方の目標を同時に狙って、寄与の大きいテクスチャから段階的に解像度を下げる縮小計画を自動作成します。テクスチャを最小(縮小フロア)まで下げても上限に届かない場合(展開後がメッシュ・ブレンドシェイプ支配のときなど)は、確認ダイアログの見出しと調整後の見込みで「テクスチャ縮小だけでは上限に届きません(Quest除外・メッシュ削減・ブレンドシェイプ整理の併用が必要)」と正直に表示し、収まると誤認させないようにしました。
- 衣装・トグル整理に、各トグルグループを「非表示固定」にしたときに減る専有アセット(そのサブツリー内のレンダラーだけが使い、外では一切使われないテクスチャ・メッシュ)の削減量チップ(圧縮後 -◯MB / 展開後 -◯MB)を追加しました。共有アセットしか持たないグループは「単独では減らない(共有テクスチャのみ)」と明示します。トグル一覧を専有削減量(展開後)の大きい順に並べ替えるオプションと、上限超過時に削除候補を大きい順に提示するガイドも追加しました。ガイドの合計は親子(入れ子)グループの専有アセットを二重計上しません(親を非表示にすれば子のサブツリーも一緒に消えるため、重複するグループはカバー合計に足しません)。

## [1.4.2] - 2026-07-18
### Fixed
- Quest変換後に顔が真っ白（のっぺらぼう）になる問題を修正しました。原因は「半透明の自動再現」が顔の基底（素肌）マテリアルを半透明と判定し、明るいアルベドのパーティクルシェーダー（加算）へ変換していたことでした。顔（VRCAvatarDescriptorのビセームメッシュ／まばたきメッシュ、または名前がface・顔・head等のメッシュ）の基底マテリアルは、透過の扱い（乗算/加算で再現・非表示・不透明化）によらず常に不透明（Toon Standard）として変換して保護するようにしました。チーク・涙・ハイライト・メイク・そばかす・頬影などのオプション重ね（名前で判定）は従来どおり乗算/加算で近似します。既存の髪・大型メッシュ保護と同じ仕組みで、非表示（最軽量）モードでも顔の基底は消えず不透明として残ります（顔が消えるのも真っ白になるのと同じく事故のため）。マテリアル別の変換方法を手動指定している場合は、その指定が常に優先されます。

## [1.4.1] - 2026-07-18
### Fixed
- SkinnedMesh統合で本ツールが作る統合先メッシュ（RARA_MergedMesh）について、ビルド時にNDMFコンソールへ出ていた次の3つの警告を解消しました。いずれも統合先の設定が不十分だったことが原因で、1.4.0以前からビルド/Play時にはずっと出ていたもの（1.4.0の「ビルド不要の即時実測」でコンソールを見る機会が増え、気づきやすくなっただけ）です。見た目・統合結果は変えずに、警告の原因だけを取り除きました。
  - 「統合先のメッシュにRoot Boneが設定されていない」: 統合先メッシュにRoot Boneを自動設定するようにしました（統合対象メッシュで最も多く使われているRoot Bone、無ければヒューマノイドのHipsボーン）。境界（Bounds）はAAOがビルド時に統合対象から自動再計算します。
  - 「統合先のメッシュにAnchor Overrideが設定されていない」: 統合先メッシュにAnchor Override（Light/Reflection Probeのアンカー）を自動設定するようにしました（統合対象で最も多く使われているAnchor Override、無ければ上記のRoot Bone）。
  - 「統合対象のSkinnedMeshRenderer間でBlendShapeの値が揃っていません」: 統合対象のメッシュ間で同名ブレンドシェイプの現在値が食い違う場合、その値を各メッシュに現在値のまま焼き込んで固定（AAO Freeze BlendShape）することで、見た目を保ったまま食い違いを解消するようにしました。ただしアニメーションで使われているシェイプや、ビセーム/まばたき/MMD標準表情モーフは誤って固定しないよう自動固定の対象外とし、その場合はレポートに注意を表示します（見た目が意図と違うときは該当メッシュを統合から除外してください）。

## [1.4.0] - 2026-07-18
### Added
- 変換直後に「ビルド不要の即時実測」を追加しました。これまで実測(アバターの本当の重さ)は ▶️(Play)か SDK の Build & Test / Upload をしないと分かりませんでしたが、Quest変換 / PC軽量化で複製を作った直後に、ビルド時とまったく同じ姿(NDMF の変換適用後・EditorOnly 除去後・Android時は非対応コンポーネント除去後)へ自動でベイクして、その場で実測レポートを表示します。Play も SDK ビルドも不要です。既定でオンで、各ツールの設定「変換直後に即時実測する(ビルド/Play不要)」でオフにできます。
- 実測レポートに「今すぐ実測(ビルド不要)」ボタンを追加しました。選択中(またはシーン上)の _Opt / _Quest 複製を、いつでもボタン一つでビルド時の姿に実測できます(「_Opt / _Quest 以外のアバターも計測する」をオンにすると、それ以外のアバターも対象になります)。
- 即時実測は PC(Windows)基準と Quest(Mobile)基準の両方を、SDK と同じ計算で測ります。実測に使った一時的な複製・生成データは計測後に完全に後始末するため、シーンやプロジェクト(Assets)には何も残りません。
- 正直な注記: 即時実測はダウンロードサイズを推定値として表示します(実際のファイルサイズは SDK ビルド/アップロード時のみ確定します)。また lilToon 等の「SDK ビルド固有の処理(コールバック)」は即時実測には含まれません(それらは従来どおり Play / 実ビルド時の実測に反映されます)。NDMF が未導入の場合、即時実測はスキップされ、従来どおり Play / ビルド時に実測されます。

## [1.3.0] - 2026-07-18
### Added
- Quest変換の対応元シェーダーを拡張しました。これまでの lilToon / NonToon に加えて、Poiyomi Toon(ロック=Lock In 済みを含む)・Unity 標準(Standard / Standard (Specular setup))・Unity Unlit 系(Unlit/Color・Texture・Transparent・Transparent Cutout)・VRM MToon(VRM0.x「VRM/MToon」/ VRM1.0「VRM10/MToon10」)を、それぞれ専用の変換経路で Toon Standard / Toon Lit へ変換します。従来これらは「メインテクスチャのみの汎用近似」に落ちて警告が出ていましたが、系統ごとにメインテクスチャ(×色)・ノーマルマップ・カリング・エミッション・マットキャップ等を適切に読み取って対応付けるようになりました(汎用近似の警告は、本当に未対応のシェーダーだけに出ます)。
- Poiyomi Toon: アルベド(メイン × カラー、Main Color Adjust の色相/彩度/明度/ガンマ補正)をベイクし、ノーマルマップ・エミッション・カリングを対応付けます。ロック済みマテリアルはタグから元テクスチャを復元して読み取ります。影ランプ・マットキャップ・リム等の一部の質感はシェーダー内部情報が参照できないため既定値へ縮退し、その旨をレポートします。
- Unity 標準 / Unlit: メインテクスチャ × カラーをベイクし、Standard はノーマルマップを引き継いで既定の2バンド影ランプ、Unlit は陰影を持たないためフラット(均一)ランプで変換します。メタリック・スムースネスは Toon Standard に相当機能が無いため反映されません。金属パーツはマテリアル別指定の「MatCap Lit(金属向け)」に切り替えると映り込みで近似できる旨を案内します。
- VRM MToon: メインテクスチャ × カラーをベイクし、ノーマルマップ・カリングを対応付けます。加算スフィア(_SphereAdd / MToon10 の _MatcapTexture)は Toon Standard の加算マットキャップへ引き継ぎ、トゥーン影(Shade 色・Toony・Shift)は Toon Standard に色付き影枠が無いため既定ランプで近似します。アウトラインは Quest 非対応のため失われます。半透明・カットアウトは _BlendMode で正しく判定します(RenderType タグが Opaque 固定で信頼できないため)。
- 上記いずれの新系統も、半透明(アルファブレンド)マテリアルは既存の透過処理設定(再現=乗算/加算 / 非表示 / 不透明化)に従い、マテリアル別の変換方法指定(Toon Standard / Toon Lit / MatCap Lit / パーティクル(加算・乗算) / 非表示 / 変換しない)も利用できます。エミッションは既存の汎用ベイク経路でメインテクスチャへ焼き込む近似です(この波ではシェーダー固有の追加GPUベイクは行いません)。

## [1.2.0] - 2026-07-18
### Added
- PC/Quest間で揺れ物を掴んだときの同期ズレを防ぐNetwork ID自動割り当てを追加しました。Questコンバーターの「_Quest」複製とPC軽量化の「_Opt」複製に、元アバターを基準にした一貫したNetwork ID(PhysBoneなどの揺れ物)を書き込むため、同じ揺れ物が両プラットフォームで同じIDになります。これにより、他ユーザーから見た掴み・ポーズ・ストレッチの同期がPC版とQuest版でズレる問題を防げます。既定でオンで、各ツールの詳細設定(Questコンバーター)/実行セクション(PC軽量化)の「Network IDを割り当てる(PC/Quest間の揺れ物の掴み同期)」でオン/オフできます。元アバターに既にNetwork IDが割り当てられている場合はそのIDを引き継ぎます。

## [1.1.0] - 2026-07-18
### Added
- lilToonのマットキャップ(金属・宝石等の映り込み)をToon Standardへ自動で引き継ぐようにしました(既定オン。変換設定詳細の「マットキャップを引き継ぐ」で切り替え可能)。マットキャップテクスチャ・合成強度・合成モード(乗算のみ乗算・それ以外は加算近似)・合成マスクを対応付けます。従来はマットキャップは変換されず警告のみでしたが、Toon Standardのネイティブマットキャップで映り込みが維持されます(独自UV・視差・ディテール法線・その2(2nd)マットキャップは非対応)。アトラス統合では、異なるマットキャップが1マテリアルに混ざらないよう同一マットキャップのマテリアルだけを統合し、マットキャップマスク付きは正しさを優先して統合対象から外します(PC最適化のアトラスでも異なるマットキャップは統合しません)。
- マテリアル別の変換方法に「MatCap Lit(金属向け)」を追加しました。金属パーツ向けの VRChat/Mobile/MatCap Lit へ変換する選択肢で、メインテクスチャに映り込み(マットキャップ)を乗算する軽量シェーダーです。元マテリアルのマットキャップ画像があれば引き継ぎ、無ければ白(素のライティング相当)で変換して割り当てを案内します。不透明のみ・乗算合成のマットキャップで、アトラス統合の対象外のためスロットを1つ使います。
### Fixed
- 変換でポリゴン数が増える問題を完全に解消しました。マテリアルスロット数がサブメッシュ数を超えるメッシュ(多重描画スロット)で、1.0.8 は「見た目に効く上描き」を専用サブメッシュへ実体化していたため三角形が水増しされていました(Kei髪の実測例: 68,674 → 78,948)。1.0.9 は三角形の複製を全廃し、上描きスロットを内容で振り分けます。何も描かない非表示(不可視)の上描き(Questで消える疑似影 FakeShadow など)は削除し、基底と同じ材質・同じアトラスセルに重なる冗長な上描きも削除します(いずれもポリゴン増ゼロ)。それ以外の実際に見える上描きは、元アバターと同じネイティブ多重描画のまま保持し(複製しない)、基底サブメッシュのUVを再配置せず上描きと同じUVを共有させることで見た目を維持します。結果、Kei髪は Quest 版=1スロット(疑似影を削除)・PC 版=元と同じ2スロット(髪本体＋疑似影がそのまま描画)となり、どちらも 68,674 のまま増えません。PC最適化・Quest変換の両アトラス経路に適用。
- 上記の変更に伴い、生成するアトラスメッシュ/アトラスアセットの命名規則を更新しました。同じアトラス配置でも 1.0.8 と 1.0.9 では別名のアセットとして出力されるため、シーンに残った 1.0.8 の複製(アップロード直前アバター等)が、内容の異なる同名アセットで上書きされて壊れることはありません(旧複製は旧アセットのまま整合して残ります)。

## [1.0.8] - 2026-07-17
### Fixed
- 変換後にポリゴン数が増える問題を修正。マテリアルスロット数がサブメッシュ数を超えるメッシュ(多重描画スロット)で、旧実装が余剰スロットを一律に実サブメッシュへ複製していたため三角形が水増しされていました(実測例: 68,674 → 78,948)。新実装では、基底と同一マテリアル/同一アトラスセルへ写像される冗長スロットは複製せず統合し(ポリゴン増ゼロ)、見た目に効く真の上描きスロットのみ専用サブメッシュ化して正しいUVを与えます。PC最適化・Quest変換の両アトラス経路に適用。
- 設定が別プロジェクト・別アバターへ漏れる問題を修正。EditorPrefs はマシン全体で共有されるため、旧ツールからの設定引き継ぎで別アバター固有のパスリスト(トグル選択・PhysBone保持リスト等)が新規プロジェクトへ混入し、「トグルパスが見つかりません」警告が大量に出ていました。設定キーをプロジェクト単位で分離し、引き継ぎはアバター非依存の共通設定のみに限定。パス不一致の警告は1件のまとめメッセージに集約しました。
- PCアトラス化で顔・体・髪のテクスチャが過度に縮小される問題を修正(実測例: 顔が1024アトラス内で128pxまで縮小)。品質フロア(顔/体/髪 ≥512px・その他 ≥256px・色のみ 16px)を導入し、フロアでも1枚に収まらない場合はアトラスを複数枚に自動分割して品質を守ります。PC側のアトラスサイズ選択に 4096 を追加(既定は 2048 のまま)。
