# Changelog

## [1.10.0] - 2026-07-20
### Added
- SkinnedMesh統合で、マテリアルプロパティアニメーション(`material._UseEmission` などのエミッション切り替え・色替え演出)の波及ガード(1.7.1)によって統合対象から外されたメッシュを、**そのマテリアルアニメーションを無効化することで統合できる**ようにしました。1.7.1 では、統合対象のメッシュ間で `material.` アニメの対象が食い違う場合、AAO(Avatar Optimizer)がそのアニメを統合先メッシュ全体へ波及させて意図しない部分にも適用してしまうのを防ぐため、アニメが向いているメッシュを統合から自動除外していました(統合プレビューに「マテリアルアニメーションが統合先全体へ波及するため統合しません」と表示)。そのため、例えばシャツ・ネクタイにエミッション切り替えギミック(`_UseMain2ndTex` / `_UseEmission` 等)を持つアバターでは、それらのメッシュが統合されず SkinnedMesh 数が減らないことがありました。1.10.0 からは、こうしてガードで除外された各行に **「マテリアルアニメーションを無効化して統合(この切り替え演出は動かなくなります)」チェックボックス**を追加しました(PC軽量化・Quest対応・統合の全ウィンドウ。既定はオフ)。オンにすると、そのレンダラーに向いた `material.*` アニメーションを **複製側のアニメーションクリップからのみ無効化(除去)**した上で統合します。無効化により波及の懸念がなくなるため、そのメッシュを統合先へまとめられます(統合後の SkinnedMesh 数が減ります。例: Quest の Poor 上限 SkinnedMesh 2 に収めたいとき、顔以外がシャツ・ネクタイのエミッション演出で分かれて 4 になっていたものを、両方チェックして 2 にできます)。チェックした行は理由が「マテリアルアニメーションを無効化して統合します」に変わり、統合後(概算)の枚数へ反映されます。
- マテリアルアニメーションの無効化は **複製に対してのみ**行い、元アバターやそれと共有するアニメーションクリップ・コントローラーは一切書き換えません(衣装・トグル整理の m_IsActive 無効化と同じ、クローン専用の複製・差し替え〈AnimatorOverrideController〉の仕組みを共用します)。対象は、デスクリプターのアニメーションレイヤーと子コンポーネント(子 Animator / MA Merge Animator 等)が参照するコントローラーの、そのレンダラーに向いた `material.*` フロートカーブのみで、他のメッシュのアニメや同じクリップ内の別バインディングは残します。変換レポートには、無効化したレンダラーごとに「'X': マテリアルアニメーションを無効化して統合しました(無効化: `_UseEmission` ほか n件 / クリップ: A, B)。該当の切り替え演出は動かなくなります。」を出します。
- マテリアル**差し替え**アニメーション(`m_Materials.Array.data[N]` によるマテリアルスロットそのものの差し替え)は、スロット番号に依存してアニメが組まれているため従来どおり統合の対象外(ブロック)のままとし、このチェックボックスは出しません(統合プレビューには引き続き「マテリアル差し替えアニメがあるためスロット削除不可」等を表示します)。無効化して統合の対象になるのは、あくまで `material.`(マテリアルのプロパティ値)を動かすアニメーションです。PC軽量化・Quest対応の両パイプラインで有効です。

## [1.9.0] - 2026-07-19
### Added
- SkinnedMesh統合で、多重描画(マテリアルスロット数 > サブメッシュ数)のレンダラーを、上描き(余剰)スロットを削除して統合できるようにしました。1.7.1 では多重描画のレンダラーを一律に統合対象外にしていましたが(統合すると AAO がビルド時に多重描画を実体化してポリゴン数が増えるため)、そのぶん統合後の SkinnedMesh 数が減らないことがありました(例: 髪が `mat_hair` + `FakeShadow` の2スロット・1サブメッシュのとき、統合後の概算が本来 2 のところ 3 に留まる)。1.9.0 からは、余剰スロットのうち **何も描かないもの(スロットが空、または Quest 対応で非表示化される疑似影 `FakeShadow` / アウトラインのみなど)は、SkinnedMesh統合の直前に複製から自動で削除します**。削除すると多重描画でなくなるため、そのレンダラーはポリゴン数を増やさずに(見た目も変えずに)通常どおり統合されます。これにより、Quest 対応では疑似影付きの髪などが **ユーザーの操作なし・見た目の変化なしで自動的に統合できる**ようになりました。変換レポートには「'X': 何も描かない上描きスロットを削除しました(統合可能になりました)」を出します。
- 見える上描き(前髪の影など、実際に重ねて描かれている余剰スロット。例: PC の `FakeShadow`)については、統合プレビューの各行に **「上描きスロットを削除して統合(前髪影などの重ね描き効果が消えます)」チェックボックス**を追加しました(PC軽量化・Quest対応・統合の全ウィンドウ。既定はオフ)。オンにすると、変換時にその上描きスロットを複製から削除して統合します。前髪の影などの重ね描き効果は失われますが、そのぶん SkinnedMesh 数・スロット数を確実に減らせます(効果を残したい場合はオフのまま=従来どおり統合対象外)。チェックすると行の理由が「上描きスロット削除により統合します」に変わり、変換レポートには「'X': 上描きスロットを削除して統合しました(重ね描き効果は失われます)」を出します。
- マテリアル差し替えアニメーション(`m_Materials.Array.data[N]` によるスロット差し替え)の対象になっているレンダラーは、上描きスロットを削除するとスロット番号がずれてアニメが壊れるため、上記の自動削除・チェックボックスのいずれの対象にもしません(統合プレビューには「マテリアル差し替えアニメがあるためスロット削除不可」と表示します)。削除するのは常にサブメッシュ数を超える余剰スロット(index >= subMeshCount)だけで、基底のサブメッシュスロットには手を付けません。処理はすべて複製に対してのみ行い、元アバターは一切変更しません。PC軽量化・Quest対応の両パイプラインで有効です。

## [1.8.1] - 2026-07-19
### Fixed
- 変換済みアバターを再処理(再変換 / 再軽量化)したときに、前回の変換で生成した統合先(`RARA_MergedMesh`、グループ指定モードでは `RARA_MergedMesh_G{n}`)の残骸が SkinnedMesh統合のプレビューと結果に悪影響を与えていた問題を修正しました。本ツールの統合先は AAO(Avatar Optimizer)がビルド時にメッシュを注入する設計のため、保存された複製プレファブ上ではメッシュ未設定(sharedMesh が空)のまま残ります。この統合先を含むアバター(1.8.0 以前のモバイル対応前フローで変換したものなど)をもう一度処理すると、(1) 統合プレビューで残骸が「メッシュが未設定のため統合対象外」として1枚分カウントされ、統合後(概算)の枚数が実際より1多く表示される(例: 素体(顔保護) + 新しい統合先 + 前回の残骸 = 3 と出るが、正しくは 2)、(2) 残骸が古い AAO Merge Skinned Mesh コンポーネント(古いソース参照付き)を保持したままだと、ビルド時に新旧2つの統合先が同じソースを取り合う二重統合の衝突を起こす、(3) メッシュ未設定でも空の SkinnedMeshRenderer は SkinnedMesh 数へ数えられる、といった問題がありました。1.8.1 からは、SkinnedMesh統合を行う直前に、複製上の前回の統合先の残骸(命名が `RARA_MergedMesh` 系 かつ メッシュ未設定のもの)を自動で削除します(本ツールが生成したメッシュ無しの成果物のため安全に除去でき、古い AAO コンポーネントごと消えるので二重統合を防げます)。削除は必ず複製に対してのみ行い、元アバターは一切変更しません。変換レポートには「前回の統合先の残骸(RARA_MergedMesh)を削除しました: n件」を出します。統合プレビューでも、これらの残骸は「前回の統合先(残骸)。実行時に複製から削除されます」と表示し、統合後(概算)の枚数から除外します(前述の例で 3 → 2。統合前=現在のエディタ上の数はそのまま=残骸を含めて表示します)。ユーザーが自分で作ったメッシュ未設定のレンダラー(本ツールの命名に一致しないもの)は従来どおり「メッシュが未設定のため統合対象外」と表示し、削除しません。PC軽量化・Quest対応の両パイプラインで有効です。

## [1.8.0] - 2026-07-19
### Added
- 既にQuest対応(VRChat/Mobile/Toon Standard・Toon Lit)のマテリアルも、Quest のアトラス統合の対象にしました。これまで Quest のアトラス統合は本ツールが変換したマテリアルだけを詰め込み対象にしていたため、購入した Quest 対応済みアバターや、元アバターを持たない過去の変換結果(本ツールの過去のアトラス出力 `RARA_Atlas_*` を含む。すべて既に VRChat/Mobile 系シェーダー)ではマテリアルスロットが減りませんでした(マテリアル行に「既にQuest対応/対応済」バッジが付き、アトラス対象一覧では「(対象外: 変換対象外)」と表示され、スロットが削減できない状態でした)。1.8.0 からは、これらの Quest 対応済みマテリアルもそのまま使いつつ、変換済みマテリアルとまったく同じグループ規則(シェーダーの同一性・カリング・影ランプの同一性・マットキャップの同一性〈マットキャップマスク付きは従来どおり除外〉)で1枚のアトラスへ統合し、スロットを削減します。VRChat/Mobile/Toon Standard (Outline) も非Outline版と同じグループへ入れて統合します(統合結果は非Outline版へ揃えます)。MatCap Lit・パーティクル系のモバイルシェーダーは統合不可のため従来どおり対象外です(アトラス対象一覧に「(対象外: MatCap Lit/パーティクルはアトラス不可)」と表示)。既存のガード(マテリアル差し替えアニメで参照される・MA の Material Setter / Material Swap が参照する・UV が 0..1 範囲外〈タイリング〉・マテリアルごとのアトラス除外指定)はそのまま適用されます。元のマテリアルアセットは一切書き換えず、統合されたぶんだけクローンのレンダラースロットが生成アトラスマテリアルへ差し替わります。変換レポートには「既にQuest対応のためそのまま使用し、アトラス統合の対象にしました: n件」を出し、入力に過去のアトラス出力(`RARA_Atlas_*`)が含まれるときは「元アバターから変換し直すとより効率的に統合できます」と案内します(アトラス同士の再統合も可能ですが、元アバターからの変換のほうが無駄なく詰められます)。この変更は Quest のアトラス統合のみで、PC のアトラス統合の挙動は変更していません。

## [1.7.1] - 2026-07-19
### Fixed
- SkinnedMesh統合で、マテリアルスロット数がサブメッシュ数を超えるレンダラー(多重描画。例: 髪が `mat_hair` と `FakeShadow` の2スロット・1サブメッシュで、影を落とすために意図的にアトラス化せず残しているもの)を、統合対象から自動で除外するようにしました。これまではこうしたメッシュも統合ソースに含めていましたが、AAO(Avatar Optimizer)はビルド時に統合の過程で多重描画を実体化(サブメッシュを複製)するため、統合するとビルド後のポリゴン数が増えていました(AAO の警告「複数マテリアルによるマルチパスレンダリング…パフォーマンスランクにおけるポリゴン数が増えるなどの影響」)。実例では PC の `_Opt` 複製がビルド時に 68,674 → 78,948 三角形へ増え、PC の 70,000 の崖(Good/Medium/Poor 共通で、超えると段差なく即 Very Poor)を越えていました。1.7.1 からは、多重描画のレンダラーを統合せずそのまま残す(=ネイティブの多重描画のまま。SDK のポリゴン数は増えない)ようにし、統合プレビューにも「多重描画(スロット数>サブメッシュ数)のため統合しません(統合するとビルド時にポリゴン数が増えるため)」と理由を表示します。エディタ側で多重描画を残す方針(1.0.9)を、ビルド時のマージが打ち消さないようにするガードです(Quest 複製は隠しスロットが既に落ちていることが多く、通常は影響しません)。手動での個別除外(「このメッシュは統合しない」)はこれまで通り効きます。
- SkinnedMesh統合で、統合対象のメッシュ間で `material.`(マテリアルプロパティ)アニメーションの対象が食い違う場合、そのアニメが向いているメッシュを統合から自動で除外するようにしました。統合すると AAO がそのマテリアルアニメを統合先メッシュ全体へ波及させ、意図しない部分にもアニメが適用されることがあるためです(AAO の警告「統合対象のメッシュ間で `_UseMain2ndTex`, `_UseEmission` へのアニメーションが異なります…意図していない部分にアニメーションが適用されてしまう可能性」)。統合ソース全員が同じ `material.` アニメ集合を持つ場合(=波及しても全員同じ)は、これまで通り統合します。除外したメッシュは統合プレビューに「マテリアルアニメーションが統合先全体へ波及するため統合しません」と表示します。
- SkinnedMesh統合の同名ブレンドシェイプ値の不一致について、アニメーションで動かしているシェイプ(1.4.1 で自動固定の対象外にしているもの)の変換レポートの案内を補強しました。アニメーションで動かしているシェイプ(例: ジャケットのトグルで動く `shrink_for_jacket`)は実行時にアニメーションが値を決めるため、統合対象間で現在値が食い違っていても多くの場合は無害です。案内に「アニメーションで動かしているシェイプは実行時にアニメーションが値を決めるため、この不定は多くの場合無害です。見た目が意図と異なる場合のみ、該当メッシュを統合から除外してください」と明記し、不要に統合を諦めないようにしました(1.4.1 の自動固定は PC軽量化・Quest対応の両方で走ります)。

## [1.7.0] - 2026-07-19
### Changed
- PhysBone一覧の残す/削除チェックボックスを全ウィンドウで「チェック=残す(稼働)」に統一しました。統合ウィンドウ(Avatar Studio)はこれまで「チェック=削除」で、他のウィンドウ(PC軽量化・Quest対応)の「残す」と逆向きだったため、迷いやすいポイントでした。今回、統合ウィンドウのPhysBone行のチェックを「残す」へ反転し(既定はチェック=すべて残す、の安全側)、列の呼称も「残す」に揃えました。これは表示だけの反転で、保存される選択データ(どのPhysBoneを削除するか)の意味は変わりません。
- PhysBoneの上限表示を対象(PC / Quest)に合わせて正しくしました。統合ウィンドウの「優先度で自動選択」ボタンは、これまで対象に関係なくQuestの上限(8本)で絞っていましたが、PC単独のときは目標PCランクのコンポーネント上限(Excellent 4 / Good 8 / Medium 16 / Poor 32)で、Quest対象のときはQuest上限(8本)で絞るようにし、ボタン名と説明にどちらの基準かを明記しました。SkinnedMesh統合が無効のときの案内(「2まで削減」)も、PC(上限 Excellent 1 / Good 2)とQuest(Poor上限 2)のどちらの基準かを対象に応じて明示します。Quest上限に関する注記は、PC単独(Quest非対象)では意味がないため隠すようにしました。
- 各PhysBoneセクションに、対象の上限の要点を1行で表示するようにしました。PCは「目標ランクの上限 / Poor上限」と、超過するとVery Poor(PhysBone・コンタクト・コンストレイントが全停止し得ます)になる点を、QuestはPoor上限を超えると分野単位で揺れものが全停止する点を明示します(みんなが躓きやすい上限の崖を見えるようにするため)。
- 三角数(ポリゴン)の上限がPCの Good / Medium / Poor いずれも70,000で共通なため、超過するとランクの段差なく即Very Poorになる「崖」を、PC診断表(単体PC軽量化ツール)と統合ウィンドウのPC列に脚注・注記として明示しました。
- 統合ウィンドウのQuest詳細設定に「マットキャップを引き継ぐ」設定を追加しました(単体のQuest対応ツールと同じ設定に連動)。オフにするとマットキャップを引き継ぎません(Quest版では削除相当)。映り込みが乱れる・エラーが出る場合はオフにすると安定することがあります。Toon Standard 変換時のみ表示します。
- AAO(Trace and Optimize)を複製へ付与するトグルを「⑦実行」の実行オプションに一本化しました。これまで「隠れメッシュ削除」パネルにも同じトグルがあり重複していたため、そちらは案内文(「AAOの付与は⑦実行の設定に統合しました」)に置き換えました(どちらも同じ設定を操作していたため、動作は変わりません)。
- プリセットのボタン列のすぐ下に、選び方の1行ガイド(「Questだけ→おまかせQuest対応 / PCも軽くする→フル両対応 / PCのみ→PCをGood(またはPoor)に」)を追加しました(詳しい説明は⑧ヘルプに引き続きあります)。
- 「目標達成ガイド」の三角数(ポリゴン)超過アドバイスを、PC・Questで出し分けるようにしました。PC単独での超過には、Questでしか使えない「Quest除外」を出さず、「ポリゴン削減(Meshia連携)セクションで削減できます(削減はビルド時)」と案内します。Quest対象での超過は従来どおりQuest除外・Meshia連携・AAO隠面メッシュ削除を案内します。
- 単体のQuest対応ツールの「PhysBoneをマージして削減」トグルを、「9. 変換設定」から「4. PhysBone設定」セクション(マージ結果のプレビューの直近)へ移設しました(単体PC軽量化ツールと同じ配置)。プレビューを見ながら切り替えられ、切り替えると即座にプレビューへ反映されます。あわせて、これまでプレビュー側に出していた「マージ設定は9.変換設定にあります」という案内は不要になったため削除しました。
- 単体PC軽量化ツールの「Network IDを割り当てる」トグルのツールチップに、これがPC版とQuest版を対にして使うための設定であり、Quest版を作らないPC単独運用では効果がない旨を追記しました。ドキュメントのメニュー項目の記述(実在するのは「RARA/アバター軽量化・Quest・iOS対応ツール」の1つ)と、Network IDトグルの場所(単体Quest対応ツールの詳細設定・単体PC軽量化ツールの実行セクション。統合ウィンドウの詳細設定には無く常にオン)の記述を実態に合わせて修正しました。

- 変換でポリゴン数が増える問題を完全に解消しました。マテリアルスロット数がサブメッシュ数を超えるメッシュ(多重描画スロット)で、1.0.8 は「見た目に効く上描き」を専用サブメッシュへ実体化していたため三角形が水増しされていました(Kei髪の実測例: 68,674 → 78,948)。1.0.9 は三角形の複製を全廃し、上描きスロットを内容で振り分けます。何も描かない非表示(不可視)の上描き(Questで消える疑似影 FakeShadow など)は削除し、基底と同じ材質・同じアトラスセルに重なる冗長な上描きも削除します(いずれもポリゴン増ゼロ)。それ以外の実際に見える上描きは、元アバターと同じネイティブ多重描画のまま保持し(複製しない)、基底サブメッシュのUVを再配置せず上描きと同じUVを共有させることで見た目を維持します。結果、Kei髪は Quest 版=1スロット(疑似影を削除)・PC 版=元と同じ2スロット(髪本体＋疑似影がそのまま描画)となり、どちらも 68,674 のまま増えません。PC最適化・Quest変換の両アトラス経路に適用。
- 上記の変更に伴い、生成するアトラスメッシュ/アトラスアセットの命名規則を更新しました。同じアトラス配置でも 1.0.8 と 1.0.9 では別名のアセットとして出力されるため、シーンに残った 1.0.8 の複製(アップロード直前アバター等)が、内容の異なる同名アセットで上書きされて壊れることはありません(旧複製は旧アセットのまま整合して残ります)。

## [1.0.8] - 2026-07-17
### Fixed
- 変換後にポリゴン数が増える問題を修正。マテリアルスロット数がサブメッシュ数を超えるメッシュ(多重描画スロット)で、旧実装が余剰スロットを一律に実サブメッシュへ複製していたため三角形が水増しされていました(実測例: 68,674 → 78,948)。新実装では、基底と同一マテリアル/同一アトラスセルへ写像される冗長スロットは複製せず統合し(ポリゴン増ゼロ)、見た目に効く真の上描きスロットのみ専用サブメッシュ化して正しいUVを与えます。PC最適化・Quest変換の両アトラス経路に適用。
- 設定が別プロジェクト・別アバターへ漏れる問題を修正。EditorPrefs はマシン全体で共有されるため、旧ツールからの設定引き継ぎで別アバター固有のパスリスト(トグル選択・PhysBone保持リスト等)が新規プロジェクトへ混入し、「トグルパスが見つかりません」警告が大量に出ていました。設定キーをプロジェクト単位で分離し、引き継ぎはアバター非依存の共通設定のみに限定。パス不一致の警告は1件のまとめメッセージに集約しました。
- PCアトラス化で顔・体・髪のテクスチャが過度に縮小される問題を修正(実測例: 顔が1024アトラス内で128pxまで縮小)。品質フロア(顔/体/髪 ≥512px・その他 ≥256px・色のみ 16px)を導入し、フロアでも1枚に収まらない場合はアトラスを複数枚に自動分割して品質を守ります。PC側のアトラスサイズ選択に 4096 を追加(既定は 2048 のまま)。
