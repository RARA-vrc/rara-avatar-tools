# RARA Avatar Tools (アバター軽量化・Quest/iOS対応)

## 概要

VRChatアバターの軽量化とQuest/iOS対応を支援するUnityエディタ拡張ツールです。すべての操作は1つの統合ウィンドウ(Avatar Studio)で行います。Unity のメニューには次の項目が追加されます。

- RARA / アバター軽量化・Quest・iOS対応ツール: 統合ウィンドウを開く(上部のチップで対象を PC / Quest(または両方)に切り替えて使います)
- RARA / 実測レポート: ビルド/Play 時の実測レポートウィンドウを開く(後述)

内部的には PC最適化エンジン(PC Optimizer)と Quest/iOS変換エンジン(Quest Converter)を共有しています。各機能の詳細は Docs フォルダを参照してください。

## AAO（Avatar Optimizer）との連携について

本ツールは anatawa12 氏の AAO: Avatar Optimizer と処理を分業する設計です。実行時は、生成した複製にのみ次のAAOコンポーネントを自動で追加・設定します（元アバターには一切追加しません）。

- Trace and Optimize … アバターを走査して自動でできる限りの最適化を行うコンポーネント（未使用ボーンの削減や自動マージなど）。追加の自動最適化のために付与します（下記の Merge Skinned Mesh / Remove Mesh By BlendShape は Trace and Optimize が無くてもビルド時にAAOが適用します）。
- Merge Skinned Mesh … 顔以外の複数の SkinnedMeshRenderer を1つの SkinnedMeshRenderer へ統合します（「SkinnedMesh統合」の実体）。統合時は BlendShape 名を自動変更して重複を避けます。レンダラーをグループ1〜8に振り分けて、まとまりごとに統合するグループ指定モードもあります（顔は常に自動保護）。
- Remove Mesh By BlendShape … 指定した BlendShape で動く頂点とポリゴンを削除します（服の下に隠れる肌などを消す「隠れメッシュ削減」の実体）。

PhysBone のマージ（設定が一致する兄弟チェーンを1本へまとめてコンポーネント数を減らす処理）は本ツール独自の実装で、AAO の Merge PhysBone は使用していません。

適用タイミング: これらの統合・削除は、アップロード/Play時のビルド（NDMF）で実行されます。そのため、生成直後の複製やエディタ上の診断値には反映されず、実際のアップロード後にさらに軽くなります。最終的な数値は VRChat SDK のビルド結果、または本ツールの実測レポート（後述）で確認してください。

AAO未導入時の挙動: AAO が無くても本ツール自体は動作します。AAO を使う機能（SkinnedMesh統合・隠れメッシュ削減など）は自動でスキップされ、ウィンドウ内に導入の案内が表示されます。導入は次のリポジトリを VCC / ALCOM に追加して行います。

- 導入リポジトリ: https://vpm.anatawa12.com/add-repo
- 公式ドキュメント: https://vpm.anatawa12.com/avatar-optimizer/ja/

謝辞: AAO: Avatar Optimizer（MIT License / anatawa12 氏）に感謝します。あわせて VRChat SDK・lilToon・NonToon・Poiyomi・Modular Avatar / NDMF にも感謝します。また、Quest/iOS変換のポリゴン削減は、1.6.0 から Meshia Mesh Simplification(MIT License / Ram.Type-0 氏)へ連携する方式へ移行しました(自前の頂点間引き実装は廃止)。

## NDMFコンソールのAAO警告・最適化ログについて

変換直後の即時実測（1.4.0 以降）や ▶️（Play）・SDK ビルドを行うと、NDMF のコンソールに AAO（Avatar Optimizer）発の警告が出ることがあります。SkinnedMesh統合（Merge Skinned Mesh）まわりで代表的なのは、統合先（統合ターゲット）に Root Bone が設定されていない・統合先に Anchor Override（アンカー上書き）が設定されていない・統合ソース間で同名の BlendShape が違う値を持つ（値不一致）の3種です。いずれも 1.4.0 以前からビルド時には発生していたもので、即時実測で毎回コンソールを通るようになって見えやすくなっただけで、新しい不具合ではありません（PC軽量化・Quest対応のどちらも、本ツールが作る統合先メッシュ RARA_MergedMesh について出ます）。

1.4.1 では、このうち機械的に解消できるものを変換側で自動処理します。統合先（Merge Skinned Mesh のターゲット）へ Root Bone と Anchor Override を自動設定して統合ソースの基準へ揃え、食い違い警告を抑えます。また、アニメーションで使われていない同名 BlendShape の値衝突は、現在の値でメッシュへ焼き込んで自動固定します（現在値のまま焼くので見た目は変わりません）。アニメーションで動かしている BlendShape は勝手に固定すると表情などが壊れるため自動固定せず、変換レポートで該当シェイプを案内します（必要なら手動で調整してください）。

1.7.1 では、統合そのものが逆効果になるレンダラーを自動で統合対象から外すガードを2つ追加しました。

- **多重描画（マテリアルスロット数 > サブメッシュ数）のメッシュは統合しない**: 髪などは、影を落とすために `mat_hair` と `FakeShadow` の2スロットを1サブメッシュに重ねて多重描画していることがあります（本ツールは1.0.9からこうしたスロットをアトラス化せず意図的に残します）。ところが AAO は統合の過程でビルド時に多重描画を実体化（サブメッシュを複製）するため、**このメッシュを統合するとビルド後のポリゴン数が増えます**（AAO の警告「複数マテリアルによるマルチパスレンダリング…パフォーマンスランクにおけるポリゴン数が増えるなどの影響」）。実例では PC の `_Opt` 複製がビルド時に 68,674 → 78,948 三角形へ増え、PCの 70,000 の崖を越えて即 Very Poor になっていました。1.7.1 からは多重描画のレンダラーを統合せずそのまま残す（＝ネイティブの多重描画のままで、SDKのポリゴン数は増えない）ようにし、統合プレビューにも「多重描画（スロット数>サブメッシュ数）のため統合しません」と表示します（Quest複製は隠しスロットが既に落ちていることが多く、通常は影響しません）。
- **マテリアルアニメーションの対象が食い違うメッシュは統合しない**: 統合対象のメッシュ間で `material.`（マテリアルプロパティ）アニメーションの対象が違う場合、統合すると AAO がそのアニメを統合先メッシュ全体へ波及させ、意図しない部分にもアニメが適用されることがあります（AAO の警告「統合対象のメッシュ間で `_UseMain2ndTex`, `_UseEmission` へのアニメーションが異なります…意図していない部分にアニメーションが適用されてしまう可能性」）。1.7.1 からは、アニメの向いているメッシュを統合から外して各メッシュに閉じたままにします（統合ソース全員が同じ `material.` アニメ集合なら波及しないためそのまま統合します）。プレビューには「マテリアルアニメーションが統合先全体へ波及するため統合しません」と表示します。

なお、上記の同名 BlendShape の値不一致で「アニメーションで動かしているシェイプ」（例: ジャケットのトグルで動く `shrink_for_jacket`）については、実行時にアニメーションが値を決めるため、統合対象間で現在値が食い違っていても**多くの場合は無害**です。変換レポートの案内でもその旨を明記しています（見た目が意図と異なるときだけ、該当メッシュを統合から除外してください）。

なお、AAO はビルド時に最適化の成果（削減したポリゴン数・統合したメッシュ数など）をコンソールへ出力します。行頭に「!」アイコンが付いて表示されることがありますが、これはエラーではなく AAO による正常な成果報告です。赤字のエラーでなければ変換は成功していますので、そのまま進めて問題ありません。

1.8.0 では、**既にQuest対応（VRChat/Mobile/Toon Standard・Toon Lit）のマテリアルもアトラス統合の対象**になりました。これまで Quest のアトラス統合は本ツールが変換したマテリアルだけを対象にしていたため、購入した Quest 対応済みアバターや、元アバターを持たない過去の変換結果（本ツールの過去のアトラス出力 `RARA_Atlas_*` を含む）ではマテリアルスロットが減りませんでした。1.8.0 からは、これらの Quest 対応済みマテリアルもそのまま使いつつ、変換済みマテリアルと同じグループ規則（シェーダー・カリング・影ランプ・マットキャップの同一性）で1枚のアトラスへ統合してスロットを削減します。MatCap Lit・パーティクル系のモバイルシェーダーは従来どおり統合対象外です。過去のアトラス出力が入力に含まれるときは、変換レポートに「元アバターから変換し直すとより効率的に統合できます」と案内します（アトラス同士の再統合も可能ですが、元アバターからの変換のほうが無駄なく詰められます）。

1.8.1 では、変換済みアバターを再処理（再変換・再軽量化）するとき、前回の変換で生成した統合先（RARA_MergedMesh、グループ指定モードでは RARA_MergedMesh_G{n}）の残骸（AAO がビルド時にメッシュを注入する設計のため、複製プレファブ上ではメッシュ未設定のまま残るもの）を、SkinnedMesh統合の直前に複製から自動で削除するようにしました。これにより、統合プレビューで残骸が1枚分多く数えられて統合後の概算枚数が実際より1多く出る問題と、古い AAO Merge Skinned Mesh が新しい統合先とソースを取り合う二重統合の衝突を防ぎます。削除は必ず複製に対してのみ行い、元アバターは一切変更しません。統合プレビューでは、これらの残骸を「前回の統合先（残骸）。実行時に複製から削除されます」と表示し、統合後（概算）の枚数から除外します。ユーザーが自作したメッシュ未設定のレンダラー（本ツールの命名に一致しないもの）は削除しません。PC軽量化・Quest対応の両方で有効です。

1.9.0 では、1.7.1 で一律に統合対象外にしていた**多重描画（マテリアルスロット数 > サブメッシュ数）のレンダラーを、上描き（余剰）スロットを削除して統合できる**ようにしました。

- **何も描かない上描きは自動削除して統合**: 余剰スロットのうち、スロットが空、または Quest 対応で非表示化される疑似影 `FakeShadow` / アウトラインのみなど**何も描かないもの**は、SkinnedMesh統合の直前に複製から自動で削除します。削除すると多重描画でなくなるため、そのレンダラーはポリゴン数を増やさず（見た目も変えず）に通常どおり統合されます。これにより Quest 対応では疑似影付きの髪などが**ユーザーの操作なし・見た目の変化なしで自動的に統合**できます（例: 髪が `mat_hair` + `FakeShadow` の2スロット・1サブメッシュのとき、統合後の概算が本来 2 のところ 3 に留まっていたのが、2 になります）。変換レポートに「'X': 何も描かない上描きスロットを削除しました（統合可能になりました）」を出します。
- **見える上描き（前髪影など）はチェックで削除して統合**: 実際に重ねて描かれている余剰スロット（例: PC の `FakeShadow`）は、統合プレビューの各行の**「上描きスロットを削除して統合（前髪影などの重ね描き効果が消えます）」チェックボックス**（既定オフ）で削除して統合できます。前髪の影などの重ね描き効果は失われますが、SkinnedMesh数・スロット数を確実に減らせます。チェックすると理由が「上描きスロット削除により統合します」に変わり、レポートに「'X': 上描きスロットを削除して統合しました（重ね描き効果は失われます）」を出します。
- **差し替えアニメ対象は不可**: マテリアル差し替えアニメーション（`m_Materials.Array.data[N]`）の対象レンダラーは、上描きスロットを削除するとスロット番号がずれてアニメが壊れるため、自動削除・チェックボックスのいずれの対象にもしません（プレビューに「マテリアル差し替えアニメがあるためスロット削除不可」と表示）。削除するのは常に余剰スロット（サブメッシュ数を超える分）だけで、基底のサブメッシュスロットには手を付けません。処理はすべて複製に対してのみ行い、元アバターは変更しません。PC軽量化・Quest対応の両方で有効です。

1.10.0 では、1.7.1 の**マテリアルアニメーションの波及ガードで統合対象から外されたメッシュを、そのマテリアルアニメーションを無効化することで統合できる**ようにしました。エミッション切り替え（`_UseEmission` / `_UseMain2ndTex` 等）などのギミックを持つシャツ・ネクタイのようなメッシュは、統合対象間で `material.` アニメの対象が食い違うため 1.7.1 のガードで自動除外され（「マテリアルアニメーションが統合先全体へ波及するため統合しません」）、SkinnedMesh 数が減らないことがありました。1.10.0 からは、こうしてガードで除外された各行に**「マテリアルアニメーションを無効化して統合（この切り替え演出は動かなくなります）」チェックボックス**（既定オフ・PC/Quest/統合の全ウィンドウ）を追加しました。オンにすると、そのレンダラーに向いた `material.*` アニメーションを**複製側のアニメーションクリップからのみ無効化（除去）**した上で統合します。無効化により波及の懸念がなくなるため統合先へまとめられ、SkinnedMesh 数が減ります（例: Quest の Poor 上限 SkinnedMesh 2 に収めたいとき、顔以外がシャツ・ネクタイのエミッション演出で分かれて 4 になっていたものを、両方チェックして 2 にできます）。無効化は複製に対してのみ行い、**元アバターやそれと共有するアニメーションクリップ・コントローラーは一切書き換えません**（衣装・トグル整理の m_IsActive 無効化と同じ、クローン専用の複製・差し替えの仕組みを共用します）。チェックした切り替え演出は動かなくなります（固定表示になります）。変換レポートには「'X': マテリアルアニメーションを無効化して統合しました（無効化: `_UseEmission` ほか n件 / クリップ: A, B）。該当の切り替え演出は動かなくなります。」を出します。なお、マテリアル**差し替え**アニメーション（`m_Materials.Array.data[N]`）は従来どおり対象外（このチェックボックスは出ません）で、無効化して統合できるのは `material.`（マテリアルのプロパティ値）を動かすアニメーションだけです。PC軽量化・Quest対応の両方で有効です。

## Meshia 連携（ポリゴン削減の実体）

Meshia Mesh Simplification（Ram.Type-0 氏 / MIT）は、NDMF でビルド時（Play / アップロード）にメッシュを高品質に簡略化するツールです。1.6.0 から、本ツールのポリゴン削減はこの Meshia へ連携する方式になりました。自前の頂点間引きは廃止し、本ツールは目標三角形数の指定と「_Quest」複製への Meshia コンポーネント追加を担当します（実際の簡略化は Meshia がビルド時に実行）。導入は VPM リポジトリ https://ramtype0.github.io/VpmRepository/ を VCC / ALCOM に追加して行います（公式ドキュメント https://ramtype0.github.io/Meshia.MeshSimplification/ ）。

使い方のパターン:

- ワンボタン運用（推奨）… 目標三角形数を指定して「_Quest 複製へ Meshia を追加」を押すと、本ツールが「_Quest」複製の子オブジェクトへ Meshia のアバター単位コンポーネント（Meshia Cascading Avatar Mesh Simplifier）を追加・設定します。顔・髪を含む配分は本ツールが目標へ合わせて設定し、追加後に Meshia コンポーネント側で個別の微調整もできます。この自動付与には Modular Avatar が必要です（未導入時はパネルに導入案内が出るだけで付与されません）。
- レンダラー単位で追い込む… 崩れやすい/見せ場の部位は、追加された Meshia Cascading Avatar Mesh Simplifier で部位ごとに目標三角形数や保護設定を調整します（さらに細かく詰めたいときは、レンダラー単位の Meshia Mesh Simplifier を手動で足す方法もあります）。EditMode プレビューで品質を確認できます。
- 二重指定に注意… 同一レンダラーに、本ツールが追加した Meshia と手動で足した Meshia の両方を効かせないでください（レンダラー単位でどちらか一方にする）。両方が同じ目標へ強くかかると、ビルド後に削りすぎになります。

注意（ビルド時反映）: Meshia の削減はビルド時に反映されるため、エディタ/診断のポリゴン数には出ません（EditMode プレビューはありますが、実メッシュの差し替えはビルド時です）。最終的な三角形数は即時実測（1.4.0 以降）または実測レポート（▶️ Play / SDK ビルド）で確認してください。⑤ ポリゴン削減パネルは、手動で追加された Meshia コンポーネントを検出すると二重指定の注意を表示します。旧バージョンで保存した自前のポリゴン削減計画は 1.6.0 では適用されません。

謝辞: Meshia Mesh Simplification（MIT License / Ram.Type-0 氏）に感謝します。

## 実測レポート（ビルド/Play 時の実測）

AAO や Meshia、Modular Avatar による統合・削減はビルド（Play / アップロード）時に適用されるため、生成直後のエディタ数値には出ません。実測レポートは、その最終的な複製そのものを VRChat SDK と同じ計算で実測し、MA / AAO / Meshia 適用後・EditorOnly 除去後の本当の数値を確認できる機能です。メニューの RARA / 実測レポート から開けます。

- _Opt / _Quest 複製を ▶️（Play）すると、ビルド時と同じ前処理を通した最終複製を実測し、結果がポップアップ表示されます（PC 基準・Quest 基準の両方。総合ランク・項目別の値・目標との達成判定・元アバターとの差を表示）。
- VRChat SDK の Build & Test / Upload でビルドすると、生成されたアセットバンドルの実ファイルサイズも記録されます（Quest/Android は 10MB 上限との判定つき）。エディタの推定ではなく、アップロードされる実バンドルサイズです。
- 既定では名前が _Opt / _Quest で終わる複製だけを計測します。「_Opt / _Quest 以外のアバターも計測する」をオンにすると全アバターを対象にできます。
- 計測を止めたいときは、実測レポートウィンドウ上部の「ビルド/Play時に実測する」をオフにします（ビルドや Play の動作自体には影響しません）。Play/ビルドのたびの自動表示だけを止めたいときは「▶️（Play）・ビルドのたびに実測レポートを表示する」をオフにします。

## 導入（VCC / ALCOM）

1. 追加ページ https://rara-vrc.github.io/rara-avatar-tools/ を開き、「VCC / ALCOM に追加」ボタンを押します。VCC（VRChat Creator Companion）または ALCOM が起動し、リポジトリの追加確認画面が開きます。
2. 手動で追加する場合は、VCC / ALCOM の Settings > Packages > Add Repository に次の URL を貼り付けます。
   - https://rara-vrc.github.io/rara-avatar-tools/index.json
3. 対象のアバタープロジェクトを VCC / ALCOM で開き、Manage Project の一覧から RARA Avatar Tools を追加します。

## トラブルシューティング

Quest/iOS変換後に、見えるべきものが消えた、または消えるべきものが見えている場合は、統合ウィンドウの⑥マテリアル(Questパネル)で該当マテリアルの変換方法を選び直して再生成してください(見せたい場合は Toon Standard(不透明)や乗算・加算、消したい場合は 非表示)。急ぎの場合は、生成された _Quest 複製のマテリアルのシェーダーを Inspector で直接 VRChat/Mobile/Toon Standard などの Quest対応シェーダーへ変更しても表示できます。ただし再生成すると生成マテリアルは同じ場所へ上書きされ、この手動変更は失われるため、恒久的にはパネルでの変換方法指定をおすすめします。Quest上では VRChat/Mobile 系のシェーダーだけが正しく表示され、それ以外はフォールバックします。アップロード前に Build & Test で実機相当の見え方を確認してください。
