# RARA Quest Converter

移行のお知らせ

操作は統合ウィンドウ『RARA アバター軽量化・Quest/iOS対応ツール』へ移行しました。旧UIは メニュー RARA > 旧ツール から引き続き使えます。メニューの RARA > Quest対応コンバーター を選ぶと、統合ウィンドウが Quest 対象で開きます(変換エンジンは同じものを共有しています)。

VRChat の PC 用アバターを Quest / スマホ(Android・iOS)対応に変換する Unity Editor 拡張。
旧UIは、メニューの RARA > 旧ツール > Quest対応コンバーター(旧UI) から開く。

本ツールは Quest / スマホ対応への変換専用。シェーダーを Mobile 系へ変換し、テクスチャに質感をベイクして Android / iOS で使える形にする。PC のまま性能ランクを上げたい(Very Poor を避けたい、Good を狙いたい)場合は、変換ではなく PC 基準のランク改善を行う姉妹ツール RARA PC軽量化ツール を使うこと。両ツールは共通エンジンを持ち、PC 軽量化の結果をそのまま本ツールの Quest 変換へ引き継げる。

主な特徴

- 元のアバターは一切変更しない非破壊変換。「アバター名_Quest」の複製を生成する
- lilToon(色調補正・影・エミッション等)や NonToon(ベース質感・影ランプ)、Poiyomi(メイン色調補正・ノーマル・エミッション。ロック済みも可)の質感をテクスチャにベイクして VRChat/Mobile/Toon Standard へ変換
- 診断 → マテリアル個別設定 → 変換 → 結果チェックまでウィンドウひとつで完結
- メニューで切り替わる衣装・目・表情差分(Modular Avatar / Avatar Menu Creator 経由)も追従
- 複数マテリアルを1枚に統合するアトラス機能で、スロット数超過の Very Poor 回避を支援(実験的)
- 目標ポリゴン数に合わせてメッシュを自動で簡略化するポリゴン削減機能。既定では無効で、明示的に有効化して使う(現在のポリゴン数が目標を超えているときはセクションに有効化の案内が出る)。顔・髪は自動で強く保護する

## 導入条件

- Unity 2022.3 系(VRChat 公式指定バージョン。2022.3.22f1 で動作確認)
- VRChat Avatars SDK 3.8.1 以降(VCC / ALCOM で作成したアバタープロジェクト。3.10.4 で動作確認)
  - 3.8.1 以降が必要なのは、変換先の VRChat/Mobile/Toon Standard シェーダーがこのバージョンで追加されたため
- lilToon(変換元アバターのマテリアルが lilToon の場合のみ必要。ベイクに lilToon 付属のベイク用シェーダーを使用する。2.3.4 で動作確認)
- レンダーパイプラインは Built-in のみ対応
- AvatarOptimizer(AAO)の事前導入を強く推奨(1.9 系で動作確認)
  - ツール自体は AAO なしでも動作するが、メッシュ結合はビルド時に AAO が担当する分業設計。素体+衣装の構成はスキンメッシュ数が Android の上限(2)を超えやすく、AAO の Trace and Optimize がないと Very Poor 判定でアップロードできないことが多い。実質必須と考えてよい
  - 導入は VCC/ALCOM にリポジトリを追加して行う。ブラウザで https://vpm.anatawa12.com/add-repo を開き、VCC/ALCOM が開いたら Add Repository → プロジェクトに AvatarOptimizer を追加 → Unity へ戻る
- Modular Avatar / NDMF は必須ではない(入っていれば MergeAnimator や Material Setter も変換対象として扱う)

## 導入方法

1. 配布の unitypackage をプロジェクトにインポートする(Assets > Import Package > Custom Package)
2. ファイル一式が Assets/RARA/QuestConverter/ 以下に展開される
3. スクリプトのコンパイルが終わると、メニューバーに RARA > Quest対応コンバーター が追加される

追加の設定は不要。asmdef を使っていないため、SDK や lilToon への参照設定なしでそのまま動く。
アンインストールは Assets/RARA/QuestConverter フォルダを削除するだけでよい。

unitypackage を作る場合は、Assets/RARA/QuestConverter を右クリック > Export Package でエクスポートする。
このとき Generated フォルダ(変換結果の出力先)が含まれていれば、チェックを外して除外すること。

## 使い方

ウィンドウは上から順に進めば完了する構成になっている(1 アバター選択 → 2 診断 → 3 マテリアル設定 → 4 PhysBone設定 → 5 衣装・トグル整理 → 6 アトラス統合 → 7 メッシュ削減 → 8 Quest除外 → 9 変換設定 → 10 ポリゴン削減 → 11 実行 → 12 ヘルプ)。

1. シーンに置いたアバター(VRCAvatarDescriptor 付き)をウィンドウで選択する(選択すると自動で診断が走る)
2. 診断結果でランク・アップロード可否・推定ダウンロードサイズを確認する
3. 透過は既定で自動再現される(乗算/加算で近似)。特定のマテリアルだけ扱いを変えたいときは、マテリアル設定の「透過マテリアルの既定処理」や各マテリアルの個別指定で調整する
4. サイズが 10MB を超えそうなら、削減提案に従ってテクスチャ縮小や Quest除外を設定する
5. スロット数が超過するアバターはアトラス統合を有効にする
6. ポリゴン数が多い場合はポリゴン削減で目標ランク(7,500 / 10,000 / 15,000 / 20,000)を選び、「自動で配分計画を作成」でメッシュごとの削減目標を割り当てる(顔・髪は自動で保護される)。先にメッシュ削減(AAO連携)・衣装・トグル整理で不要なメッシュを消してから使うと画質を保ちやすい
7. 実行セクションの「この変換で行われること」サマリーを確認して「Quest対応版を生成」を実行、レポートと「変換結果チェック」(変換前後のサムネイル比較)を確認する
8. アップロード時は SDK コントロールパネルでビルドターゲットを Android に切り替えてから「_Quest」の方をアップロードする。PC 版と同じ Blueprint ID を使うと、1 つのアバターで PC / Quest 両対応になる

## できること

1. Quest適合診断
   - VRChat SDK の性能統計 API を Android/iOS 基準で実行し、総合ランクとカテゴリ別ランクを表示する
   - 非 Mobile シェーダーのマテリアル、非対応コンポーネント、Android 縮小設定のないテクスチャを一覧表示する
   - EditorOnly タグ付きのオブジェクトと Quest除外に登録したオブジェクトは、実際のビルドと同様に除外した状態で計算する
2. サイズ診断(推定)
   - 推定ダウンロードサイズを 10MB 上限と比較して表示する(テクスチャ / メッシュ / アニメーションの内訳付き。あくまで目安)
   - メインテクスチャだけでなく、エミッションマップ・ノーマルマップ・マスク類も含めた全テクスチャが対象。寄与の大きいものを上位から一覧表示し、削減提案を削減量順に表示する
   - テクスチャの提案は「適用」(個別)/「提案をすべて適用」(一括)で「縮小計画」に登録できる
   - 「10MB以下まで自動調整」ボタンで、寄与の大きいテクスチャから順に解像度を段階的に下げる計画を自動作成し、確認ダイアログで内容(どれを何pxにするか)を見てから一括登録できる。顔・体系の名前を持つテクスチャは後回しにして画質を優先する
   - 縮小計画に登録されたテクスチャは、変換時に低解像度のコピーが Generated に生成され、変換後マテリアルがそれを参照する。元のテクスチャとそのインポート設定には一切触れない。「縮小計画をクリア」でいつでも取り消せる
3. マテリアル別の変換方法指定
   - アバターの全マテリアルが一覧表示される(透過/カットアウト/パーティクル/アニメ使用/メニュー・ギミック参照などのバッジ付き)
   - マテリアルごとに変換方法を選べる: 自動(推奨) / Toon Standard / Toon Lit / パーティクル加算 / パーティクル乗算 / 非表示 / 変換しない
   - 各行に「変換後にどうなるか」の説明が表示されるので、実行前に結果を把握できる
   - Renderer 以外のコンポーネント(Modular Avatar の Material Setter、Avatar Menu Creator 等)から参照されているマテリアルも収集・変換・差し替えされる(メニューで切り替わる目・衣装・表情差分の消失を防ぐ)
4. 半透明マテリアルの自動再現(既定)
   - アルファブレンド透過マテリアル(表情用チーク、涙、ガラス等)は、Quest 版では乗算/加算のパーティクルシェーダーで半透明を近似再現する。設定不要で、チーク・涙・ガラス系がそれらしく表示される(旧バージョンの「非表示」から既定を変更)
   - 暗めの透過(チーク・頬染め・影・すりガラス等)は VRChat/Mobile/Particles/Multiply(乗算)で、明るい透過(涙・アイハイライト・光り物等)は VRChat/Mobile/Particles/Additive(加算)で近似する。メインテクスチャの明るさから自動で判定する
   - 既定処理は「3. マテリアル設定」の「透過マテリアルの既定処理」で切り替えられる: 自動で半透明を再現(推奨) / 非表示にする(最軽量) / 不透明に変換。マテリアル単位の上書きも同じ一覧で行える
   - 近似の限界: 乗算は背景を暗くする方向のみ、加算は明るくする方向のみで、色味は保持されるが(無彩色にはならない)明暗のどちらかに寄る。加算は暗いワールドで発光して見える。ちょうど中間の半透明(白いすりガラス等)は背景次第で見え方が変わる
   - 大型メッシュと髪(ポリゴン数が多い、または名前に hair/髪 等を含む)は保護対象で、既定処理に関わらず常に不透明として変換される(透過髪が消える・板化する事故を防ぐ。近似も非表示もしない)
   - カットアウト(髪の透け先端など)は形状が意味を持つため対象外で、不透明として変換される
   - 「非表示にする」を選んだ場合は、オブジェクト全体が透過なら EditorOnly 化(描画コストごと削減)、一部スロットだけ透過なら実質不可視のマテリアルに差し替えて隠す
5. 衣装・トグル整理(メッシュ/スロット削減、Quest・PC両対応)
   - トグルで切り替える衣装・アクセサリを検出して一覧表示する。各グループを「トグル維持 / 表示で固定 / 非表示で固定」から選べる
   - 「表示で固定」= 常時表示にしてトグルアニメを外す。AAO がビルド時に結合できるようになり、スキンメッシュ数・マテリアルスロット数を大きく削減できる(そのトグルは無くなる)
   - 「非表示で固定」= そのメッシュを削除(EditorOnly)。「現在の表示状態で固定(全トグル)」ワンクリックボタンあり
   - Quest 対応の仕組み上、衣装トグルを分離メッシュのまま持つとメッシュ2・スロット4の上限に入らない。ここで固定すれば上限内に収められる
   - PC のままランクを上げたい場合は、姉妹ツール RARA PC軽量化ツール を使うこと。シェーダー/テクスチャ変換をせずに、トグル整理・PhysBone マージに加えてテクスチャメモリ削減やマテリアル統合まで行い、PC(Windows)基準でランクを改善する(出力は _Opt 複製とプレファブ)。従来この機能をこのツールの「PC最適化のみ」モードとして提供していたが、PC 向けの機能はそちらへ集約した
   - 安全設計: 元アバターの共有アニメクリップは変更しない(クローン側のクリップだけ編集)。m_IsActive 以外のアニメ(ブレンドシェイプ等)は維持される
6. アトラス統合(実験的)
   - 互換性のある複数の変換済みマテリアルを 1 枚のアトラステクスチャ+1 マテリアルに統合し、メッシュの UV を再マップする
   - マテリアルスロット数と描画コストが大きく減るため、スロット数超過で Very Poor になっていたアバターを Medium 圏内に入れられる可能性がある
   - メッシュ結合そのものは AAO がビルド時に行う。本機能でスロットが同一マテリアルに揃うため、AAO の統合効果が最大化される
   - アニメーションでマテリアル差し替えされるもの、UV が 0..1 範囲外(タイリング使用)のものは自動で対象外になる(理由付きで表示)
7. メッシュ削減(AAO連携)
   - 服の下に隠れて見えない肌などのポリゴンを削減する。アバターが持つ「shrink(縮小)ブレンドシェイプ」を自動検出し、AAO の RemoveMeshByBlendShape を複製に自動セットアップできる(見えない部分だけをビルド時に削除)
   - Trace and Optimize が無ければ複製に追加してビルド時最適化を有効化する
   - shrink ブレンドシェイプが無いアバター向けに、AAO の Remove Mesh in Box(箱で範囲指定削除)/ By Mask(マスク画像で削除)の手引きを表示する
   - 顔の表情・MMD・リップシンク用ブレンドシェイプは誤検出しない安全策付き
   - SkinnedMesh統合(顔以外を1つへ): 顔(ビセーム/まばたき)以外の SkinnedMeshRenderer を AAO の Merge Skinned Mesh で 1 つへ統合し、SkinnedMesh 数とマテリアルスロット数を確実に減らす(Quest Poor 上限 SkinnedMesh 2 / スロット 4 対策)。「統合しない / 顔以外を統合(推奨)」から選ぶ。既定は統合(旧設定から引き継いだ場合は統合しない=従来動作のまま)
   - 統合するもの: 顔以外の通常の SkinnedMeshRenderer(素体・衣装・髪・アクセサリ等)。統合しないもの: 顔メッシュ(ディスクリプタの VisemeSkinnedMesh / まばたきメッシュ=分離維持)、Cloth が付いたメッシュ(布シミュレーションを保持)、EditorOnly(ビルド除外)、プレビューで個別に「統合しない」を選んだメッシュ
   - 表情は保たれる: 顔メッシュを分離維持するため、ビセーム・まばたき・表情ブレンドシェイプはそのまま動く。顔以外に残るブレンドシェイプ(靴下オフ等)・マテリアルアニメも、統合時に AAO がブレンドシェイプ名の改名・アニメの再パスを行うため追従して動き続ける(同名衝突は別名化=RenameToAvoidConflict で回避)
   - ただし表示/非表示の切り替え(トグル)は追従しない: 統合先は常時表示になり、AAO がビルド時にソースの m_Enabled / m_IsActive アニメを無効化するため、統合したメッシュの ON/OFF トグルは効かなくなる(その衣装・装飾は常時表示になる)。切り替えを残したいメッシュはプレビューで「このメッシュは統合しない」を選ぶ。常時表示でよいものは「衣装・トグル整理」で表示固定にしてから統合するとよい
   - プレビュー表: レンダラーごとに「統合する/しない+理由」と統合後の想定 SkinnedMesh 数(例 5→2)を表示する。マテリアルスロットは統合時に同一マテリアルのサブメッシュがビルド時に重複排除される(アトラスと併用でさらに削減)
   - 反映のタイミング: この統合は NDMF ビルド時(Play/アップロード)に AAO が実行する。保存された複製プレファブ自体は統合前のメッシュ・スロット数のまま表示される(既存のブレンドシェイプ削除・自動統合と同じ挙動)。最終的な数値は SDK のビルド結果で確認すること
8. ポリゴン削減(メッシュ簡略化で目標ポリゴン数へ)
   - 目標ポリゴン数(Excellent 7,500 / Good 10,000 / Medium 15,000 / Poor 20,000)を選び、「自動で配分計画を作成」でメッシュごとの削減目標を自動配分する。優先度の低いメッシュ(その他 → 衣装 → 素体 → 髪 → 顔)から順に削って目標に収める
   - 頂点を間引く方式は既存の頂点だけを残すため、UV・法線・ボーンウェイト・ブレンドシェイプ(表情)がそのまま保持され、破綻を抑えられる(新しい頂点は作らない)
   - 顔(リップシンク対象。VisemeSkinnedMesh と face/顔/head 等の名前で判定)と髪は品質下限で強く保護され、控えめにしか削減されない。各メッシュのカテゴリはバッジ(顔/髪/素体/衣装/その他)で表示される
   - メッシュごとに目標をスライダーで微調整でき、削りたくないメッシュは「除外」できる。「戻す」で自動配分値へ復帰する。計画を適用しても目標を超える場合は赤字で警告する
   - 先にメッシュ削減(AAO連携)・衣装・トグル整理で不要なメッシュを消してから使うと、同じ目標でも削る量が減って画質を保ちやすい
   - 元アバターは変更されず、削減後のメッシュは Generated 配下に生成されて複製のレンダラーに割り当てられる
   - Unity Cloth が付いたメッシュは頂点数に依存する係数配列が壊れるため、削減対象から自動的に除外される(布シミュレーションを維持するため)
9. Quest除外オブジェクト(Quest版だけの軽量化)
   - 登録したオブジェクトは _Quest 複製でのみ EditorOnly 化され、ビルドから完全に除去される。PC 版には一切影響しない
   - 残すメッシュのボーンやアイトラッキング・リップシンクの参照を巻き込む場合は警告が出る
10. Quest対応版の生成(非破壊変換)
   - マテリアル変換: lilToon の色調補正・2nd/3rd レイヤーをベイクし、Toon Standard(既定: 影ランプ・ノーマル・エミッション・最低明るさを引き継ぐ)または Toon Lit(最軽量)へ変換。エミッションの HDR 強度も2倍まで引き継ぐ
   - NonToon(jp.lilxyzw)対応: lilToon と同様に自動変換する。ベーステクスチャ・ノーマルマップを引き継ぎ、影グラデーションを影ランプテクスチャへ生成、マットキャップは近似変換する。NonToon にはエミッションが無いためエミッションは付かない。アウトライン・ファーは Quest 非対応のため外れる(透過/カットアウトはレンダリングモードで判定するため、半透明の再現・非表示化も lilToon と同じ扱いになる)
   - Poiyomi(.poiyomi/Poiyomi Toon)対応: lilToon と同様に自動変換する。メインカラー(色調補正=色相/彩度/明度/ガンマ)をベイクし、ノーマルマップ・エミッションを引き継いで Toon Standard へ変換する。影ランプは SDK 既定を使い、マットキャップ・リム・アウトライン・シャドウ調整などの追加表現は再現されない(一部の質感は既定値の近似になる)。半透明/カットアウトはブレンド設定(_SrcBlend/_DstBlend・_ZWrite・_Cutoff 等)から判定するため、半透明の再現・非表示化も lilToon と同じ扱いになる。ロック済み(Lock In / シェーダー最適化済み)のマテリアルもそのまま変換できる。ロック時に剥がされたテクスチャはマテリアルタグから復元し、焼き込まれて失われたプロパティは既定値へ縮退する(この場合は一部の質感が既定値になる旨がレポートに出る)
   - パーティクル対応: ParticleSystem / TrailRenderer / LineRenderer のマテリアルは不透明化せず Mobile/Particles 系へ自動変換。メッシュと兼用のマテリアルは 2 種類に変換して正しく割り当てる
   - アニメーション変換: マテリアル差し替えアニメ(FX 等)を検出し、コントローラーとクリップを複製して変換後マテリアルを参照させる
   - コンポーネント処理: Unity コンストレイント→VRCConstraint 変換、Cloth / Camera / Light / AudioSource / 物理 / DynamicBone / FinalIK 等の削除、設定が一致する兄弟 PhysBone チェーンのマージ(揺れを維持したままコンポーネント数を削減。既定オン)、マージ後も Poor 上限を超える場合の超過分の自動削除(任意)
   - PhysBone 設定セクション: マージ適用後の構成(髪ルート配下 12本→1、スカート配下 8本→1、マージ対象外とその理由)を変換前にプレビュー表示する
     - 既定は選択制(OptIn): すべて停止状態から、稼働させたい PhysBone をリストから選ぶ。100本超のアバターでも取捨選択しやすい(従来の「外したものだけ削除」方式にも切替可能)
     - 「優先度で自動選択」ボタン: Hair Root / Hair Front / hair side / Breast などの名前パターンで優先度を判定し、Poor 上限(8個)に収まるまで自動でチェックを入れる
     - 積極マージ(既定オン): 設定が完全一致しない兄弟チェーンも先頭の設定に統一してマージ(差異があるものは「設定差異あり」バッジ表示)。アニメ制御・パラメータ使用などの安全ゲートは維持
     - 選択後の個数が Medium 6 / Poor 8 に対してリアルタイム表示され、ランク別のコンポーネント上限ラダー(Excellent 0 / Good 4 / Medium 6 / Poor 8)と次ランクまであと何個削ればよいかの目安も併記される。8超過時は赤の強警告(モバイルは Poor 超過で全 PhysBone・コンタクト・コンストレイントが無効化されるため)。変換レポートでもエラー表示される
   - 変換できないもの: TextMeshPro は警告して Quest除外を提案。壊れたシェーダーはエラー報告してスキップ。いずれも変換全体は止めない
11. Android/iOS ビルド時の自動クリーンアップ(Auto Fix 相当)
   - FaceEmo 等がビルド時に注入する非対応コンポーネント(AudioSource 等)を、ビルド前処理で自動削除する。SDK の赤アラートで手が止まらない
   - ビルド用のコピーにのみ適用されるため、シーンのアバターは変更されない。PC ビルドでは何もしない
12. 生成物は Assets/RARA/QuestConverter/Generated/アバター名/ に保存される
   - 同じアバターを再変換すると、生成アセットは同じファイル(同じ GUID)へ内容だけ上書きされる。以前の _Quest 複製や、生成マテリアルを手動で割り当てた箇所の参照は壊れない
   - 例外はアニメーターコントローラーのみ(技術的制約で作り直しになる。旧 _Quest 複製のアニメ参照だけが切れるため、複製は新しいものに置き換えること)
   - 使われなくなった前回の生成物はレポートに一覧表示される(自動削除はしない)
   - 生成し直すと、シーンに残った古い複製(_Quest)は自動削除される。見た目が崩れた古い複製が残っている場合は削除して再生成する
   - 設定を変えて再生成するとメッシュ・アトラスは新しいアセットになり、古い複製やプレファブ参照はそのまま以前の見た目で残る(参照を新しい生成物へ差し替えるには _Quest プレファブを使う)

## ダウンロードサイズが 10MB を超える場合の削減手順

効果が大きい順。サイズ診断の削減提案にも同じ内容が具体的な数値付きで出る。

1. テクスチャの Android 最大サイズを下げる(最も効く)
   - まず全テクスチャを 1024px 制限にする。それでも超えるなら、顔・体以外(小物、衣装、マスク類)を 512px にする
2. ASTC のブロックサイズを大きくする(6x6 → 8x8 で約半分になる。画質はやや落ちる)
3. 使わないオブジェクトを Quest除外(EditorOnly)にする(予備衣装、ギミック等)
4. AAO の Trace and Optimize を有効にしておく(未使用ブレンドシェイプの固定などがビルド時に効く)
5. それでも超える場合はメッシュ側の削減を検討する(ポリゴン削減セクションで目標ポリゴン数に合わせて自動簡略化する / 不要ブレンドシェイプの削除)

## Quest / iOS の主な制限値(参考)

iOS は Quest(Android)と同じモバイル用の制限が適用される。

- ダウンロードサイズ: ビルド後圧縮 10MB まで(超過はアップロード不可)
- Very Poor はモバイルへアップロード不可。既定の表示ランクは Medium(Poor は相手が Show Avatar しないと見えない)
- ランク別の上限(抜粋): ポリゴン 7,500 / 10,000 / 15,000 / 20,000、スキンメッシュ 1 / 1 / 2 / 2、マテリアルスロット 1 / 1 / 2 / 4(Excellent / Good / Medium / Poor)
- PhysBone: コンポーネント 0 / 4 / 6 / 8、対象ボーン 0 / 16 / 32 / 64、コライダー 0 / 4 / 8 / 16、コンタクト 2 / 4 / 8 / 16
- コンポーネント数が Very Poor の閾値を超えると、モバイルでは PhysBone・コンタクト・コンストレイントがすべて無効化される

## EditorOnly と削除に関する補足

- EditorOnly タグを付けたオブジェクトはアップロードを妨げない。ビルド時に完全に除去され、サイズ・ランク計算からも外れる
- 非アクティブにするだけではビルドに含まれる。確実に外すなら EditorOnly を使う
- 手動で外す場合、アバタールートへの EditorOnly 付与と、スキンメッシュが参照するボーンの除外は避けること(本ツールの Quest除外機能はこれらを自動で検証する)

## 制限・注意

- Quest の仕様上、メッシュの透過は Particles 系シェーダーのみ。Toon Standard / Toon Lit に透明・カットアウトはない
- メッシュの半透明は Particles Additive/Multiply による近似のみ。VRChat/Mobile/Particles/Alpha Blended はアバター許可リスト外のため使用不可(実機で不透明フォールバックされる)
- アウトライン、マットキャップ、ラメ、ファーなどの高度な質感はベイク対象外(変換レポートに警告が出る)
- 前髪影(疑似影 / FakeShadow)やアウトライン専用(OutlineOnly)などの効果専用シェーダーは、Quest では再現できず不透明の白い面としてベイクされてしまうため、自動で非表示化される(髪などのベース面はそのまま変換される。個別に別の変換方法を指定した場合はそちらが優先される)
- ポリゴン削減は顔(リップシンク対象)・髪を既定で強く保護するが、目標を極端に下げると形状が崩れる。まずは Poor(20,000)を目標にし、無料の削減(メッシュ削減・衣装整理・Quest除外)を先に済ませてから使うと崩れにくい。元アバターは変更されず、削減後のメッシュは複製にのみ適用される
- リムライトは既定では変換されない(挙動の違いで想定外のハイライトが出るため)。必要なら変換設定の「リムライトを近似変換」をオン
- エミッションマップ未設定のマテリアルはエミッションなしとして変換される(単色発光を使いたい場合は生成マテリアルで手動設定)
- 診断のランクはエディタ上の状態で計算されるため、AAO 等がビルド時に統合する分は反映されず、実際より悪めに出ることがある。サイズ診断も推定値なので、最終確認は SDK のビルド結果で行うこと
- 再変換は生成アセットを GUID 保持で上書きするため参照は壊れない(アニメーターコントローラーのみ作り直しになるので、旧 _Quest 複製は新しいものへ置き換え推奨)
- Quest除外は相対パスで記録するため、同名の兄弟オブジェクトや名前に「/」を含むオブジェクトは登録できない(登録時に検証される)

## トラブルシューティング

- 髪が Quest 版で消えた: 旧バージョンの透過自動非表示が原因。最新版では大型メッシュ・髪は自動非表示の対象外になったため、再変換すれば直る
- 変換後に一部が真っ白に光る: エミッション設定の引き継ぎが原因のことが多い。最新版では自動で対処されるため、再変換すれば直る。残る場合は該当マテリアルの Emission を生成マテリアルで手動調整
- メニューで切り替える目・衣装が Quest で消える: 最新版ではメニュー/ギミック参照のマテリアルも自動変換される。マテリアル設定に「メニュー/ギミック参照」バッジ付きで表示されているか確認
- チーク等が顔に板として乗る: 既定は「透過マテリアルの既定処理 = 自動で半透明を再現」で乗算/加算に近似される。板に見える場合は、マテリアル設定でその既定処理(非表示にする 等)や個別のマテリアル指定を確認する
- SDK ビルド画面に赤いシェーダーアラートが出る: 変換前の元アバターを選んでいないか確認。アップロードするのは「_Quest」の方
- 暗いワールドで真っ黒になる: 最新版では lilToon の最低明るさを引き継ぐため再変換で改善する
- 変換したのにパフォーマンスが改善しない / PhysBone が減っていない: 変換で作った新しい「_Quest」複製ではなく、前回変換した古い複製をビルド/アップロードしている可能性が高い。最新版は再変換時に古い複製を自動削除して新しい複製をアクティブ化・ハイライト表示するので、再変換してハイライトされた複製をアップロードすること。旧版で作った古い複製がシーンに残っている場合は手動で削除する(SDK パネルの「Previous build of ...」は前回ビルドの数値なので、必ず新しい複製を選び直して確認する)
