RARA アバター軽量化・Quest/iOS対応ツール

概要

1つのウィンドウで、PC(Windows)向けの軽量化と、Quest/iOS(Android)向けの対応変換をまとめて実行します。メニューは RARA/PC軽量化ツール または RARA/Quest対応ツール から開きます。どちらも同じ統合ウィンドウが開き、上部のチップで対象を PC だけ・Quest だけ・両方に切り替えられます(「PC軽量化ツール」は PC 対象、「Quest対応ツール」は Quest 対象で開くだけの違いで、開いたあとはどちらの対象にも切り替えられます)。

元アバターは一切変更しません。実行すると、PC 用に「{名前}_Opt」、Quest 用に「{名前}_Quest」という複製がシーンとプレファブの両方に生成されます。アップロードするのはこの複製です。

AAO(Avatar Optimizer)との連携について

本ツールは anatawa12 氏の AAO: Avatar Optimizer と処理を分業する設計です。実行時は、生成した複製にのみ次の AAO コンポーネントを自動で追加・設定します(元アバターには一切追加しません)。

- Trace and Optimize … アバターを走査して自動でできる限りの最適化を行うコンポーネント(未使用ボーンの削減や自動マージなど)。追加の自動最適化のために付与します(下記の Merge Skinned Mesh / Remove Mesh By BlendShape は Trace and Optimize が無くてもビルド時にAAOが適用します)。
- Merge Skinned Mesh … 顔以外の複数の SkinnedMeshRenderer を1つの SkinnedMeshRenderer へ統合します(SkinnedMesh統合の実体)。統合時は BlendShape 名を自動変更して重複を避けます。
- Remove Mesh By BlendShape … 指定した BlendShape で動く頂点とポリゴンを削除します(服の下に隠れる肌などを消す、隠れメッシュ削減の実体)。

適用タイミング: これらの統合・削除は、アップロード/Play時のビルド(NDMF)で実行されます。そのため、生成直後の複製やエディタ上の診断値には反映されず、実際のアップロード後にさらに軽くなります。最終的な数値は VRChat SDK のビルド結果で確認してください。

AAO未導入時の挙動: AAO が無くても本ツール自体は動作します。AAO を使う機能(SkinnedMesh統合・隠れメッシュ削減など)は自動でスキップされ、ウィンドウ内に導入の案内が表示されます。導入は次のリポジトリを VCC / ALCOM に追加して行います。導入リポジトリ https://vpm.anatawa12.com/add-repo 、公式ドキュメント https://vpm.anatawa12.com/avatar-optimizer/ja/ 。

謝辞: AAO: Avatar Optimizer(MIT License / anatawa12 氏)に感謝します。あわせて VRChat SDK・lilToon・NonToon・Poiyomi・Modular Avatar / NDMF にも感謝します。

メニュー構成

RARA メニューは次の2項目だけです。どちらも同じ統合ウィンドウを、対象を絞って開きます。

- RARA/PC軽量化ツール … 統合ウィンドウを PC 対象で開く(ウィンドウ内で Quest にも切替可)
- RARA/Quest対応ツール … 統合ウィンドウを Quest 対象で開く(ウィンドウ内で PC にも切替可)

旧2ツールとの関係

このツールは、旧 PC軽量化ツール / Quest対応コンバーター と同じ変換エンジンを共有しています。中身の処理(トグル整理・SkinnedMesh統合・テクスチャ縮小・アトラス統合・PhysBone整理・シェーダー変換・ポリゴン削減など)は旧ツールと同一です。旧ウィンドウ(単体UI)はメニューからは外しましたが、クラスと Open() API はコード互換のために残しています。

このウィンドウに移植された Quest 向けパネル

旧 Quest対応コンバーターにあった Quest 固有の設定パネルを統合ウィンドウへ移植しました。

- Quest 除外(ビルド除外パス)… 構成整理ステップ
- 隠面メッシュ削減(服の下に隠れる肌などをブレンドシェイプで削除)… 構成整理ステップ
- アトラス統合 … マテリアルステップ
- 表情デカール(チーク/涙/アイハイライト)の再現・非表示 … マテリアルステップ
- Quest 変換設定(シェーダー・テクスチャ形式など)… マテリアルステップの末尾(実行の直前に最終確認)

使い方

上から順に進めれば設定できます。

1. アバターを選ぶ。シーンに置いたアバター(VRCAvatarDescriptor)を指定します。
2. 対象を選ぶ。PC / Quest のチップで切り替えます。既定は両方です。
3. プリセットを選ぶ(任意)。雑にQuest対応・PCをPoorに・PCをGoodに・フル両対応(おすすめ)から、目的に近いものを押すと対象と目標ランクがまとめて設定されます。あとから各項目で微調整できます。
4. ① 診断。現在の値を PC 基準と Quest 基準の二列で確認し、対象ごとの目標ランクを選びます。
5. ② 構成整理。トグルの固定、SkinnedMesh の統合を設定します。Quest 対象のときは、続けて Quest 除外(ビルド除外)と隠面メッシュ削減もここで設定できます。
6. ③ PhysBone。残す・削除・マージを設定します。選択後のコンポーネント数がPC/Questの上限とあわせて色付きで表示され、ランク別のコンポーネント上限ラダー(PC: Excellent 4 / Good 8 / Medium 16 / Poor 32、Quest: Excellent 0 / Good 4 / Medium 6 / Poor 8)と、次のランクまであと何個削ればよいかの目安も併記されます。Questはコンポーネント8個を超えるとモバイルで揺れものが全て停止するため、超過している場合は「優先度で自動選択(Quest 8本まで)」を押すと、髪・胸などを優先して自動的に上限内へ収められます。
7. ④ テクスチャ。縮小するテクスチャを選びます。
8. ⑤ ポリゴン。Quest 向けのポリゴン削減を設定します。ポリゴン削減は既定で無効です(明示的に有効化)。現在の三角形数が目標を超えているときは、パネルに琥珀の案内と「ポリゴン削減を有効にする」ボタンが出るので、そこから有効化します。大きく削る場合は、先に外部ツール(Blender等)でメッシュを整えてから本ツールで微調整するのがおすすめです。
9. ⑥ マテリアル。PC 列と Quest 列を並べて、アトラス統合や変換方法を設定します。Quest 対象のときは、アトラス統合・表情デカール(チーク/涙/アイハイライト)・Quest 変換設定(シェーダー・テクスチャ形式など)もこのステップで設定します。変換設定は実行の直前に最終確認できるよう末尾に配置しています。
10. ⑦ 実行。実行オプション(プレファブ保存・AAO付与)を確認し、「この実行で行われること」の要約と実行前の確認ダイアログの内容を確認して実行してください。実行後は、変換前後の比較テーブル(PC/Quest、目標との突き合わせ)と、Quest の変換結果チェック(元 ← → 変換後マテリアルのサムネイル比較)が結果欄に表示されます。
11. ⑧ ヘルプ / はじめての方へ。かんたんな説明(3行)・軽くなる仕組み(処理→何が減るか)・Quest/iOS版の割り切り・迷ったら3ステップ・4段階ランクの主要しきい値の目安をまとめています。はじめての方はここから読むと迷いにくくなります。

診断の目標達成ガイド

① 診断の下部に「目標達成ガイド」が出ます。選んだ目標ランクを超えている項目があると、その項目ごとに推奨アクションと「→ ステップnで対応」ボタンが並びます。ボタンを押すと、対応するステップ(テクスチャ・ポリゴン・PhysBone など)が開きます。

繰り返し再生成の安全性

このツールは、生成した結果を見て設定を直し、また生成する、という繰り返しを前提に作られています。

元プレファブは非破壊です。生成物は元アバターとは別の複製です。
何度でも再生成できます。実行のたびに前回の「_Opt」「_Quest」複製は自動で作り直されるため、複製が増え続けることはありません。
古い複製は自動で削除して作り直します。生成アセットは決まった場所へ上書き保存されるので、参照が壊れることもありません。
元アバターの名前を変えたあとに取り残された古い複製がある場合は、実行結果に名前付きで警告を出します(自動では削除しません)。

導入条件

AvatarOptimizer(AAO)の事前導入を強く推奨します。メッシュ・マテリアルスロットの統合や非表示メッシュの削減はビルド時に AAO が担当する分業設計のため、AAO が無いと削減効果が出にくく、Quest ではスキンメッシュ数の上限に収まらず Very Poor 判定になりやすく、既定で相手にフォールバック表示され揺れ物も停止し得ます(アップロード可否自体はサイズ上限のみで決まります)。導入は VCC/ALCOM にリポジトリを追加して行います。ブラウザで https://vpm.anatawa12.com/add-repo を開き、VCC/ALCOM が開いたら Add Repository → プロジェクトに AvatarOptimizer を追加 → Unity へ戻ってください。

注意

AAO(AvatarOptimizer の Trace and Optimize)によるメッシュやマテリアルスロットの統合、非表示メッシュの削減は、ビルド(アップロード)時に反映されます。そのため、生成直後にシーン上や診断表で見える数値は、最終的なビルド後よりも控えめ(改善が少なめ)に出ます。実際のアップロード後はさらに軽くなる場合があります。

見た目がおかしいときは(Quest/iOS)

Quest/iOS変換後に、見えるべきものが消えた、または消えるべきものが見えている場合は、⑥マテリアルのQuestパネルで該当マテリアルの変換方法を選び直して再生成してください(見せたい→Toon Standard(不透明)や乗算・加算、消したい→非表示)。非表示になった行には「不透明にする」「乗算で再現」のショートカットも用意しています。急ぎの場合は、生成された _Quest 複製のマテリアルのシェーダーを Inspector で直接 VRChat/Mobile/Toon Standard などの Quest対応シェーダーへ変更しても表示できます。ただし再生成すると生成マテリアルは同じ場所へ上書きされ、この手動変更は失われるため、恒久的な対応としてはパネルでの変換方法指定を推奨します。Quest上では VRChat/Mobile 系のシェーダーだけが表示され、それ以外はフォールバックします。アップロード前に Build & Test で実機相当の見え方を確認してください。
