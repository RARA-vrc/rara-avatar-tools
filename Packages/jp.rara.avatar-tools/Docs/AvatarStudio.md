RARA アバター軽量化・Quest/iOS対応ツール

概要

1つのウィンドウで、PC(Windows)向けの軽量化と、Quest/iOS(Android)向けの対応変換をまとめて実行します。メニューは RARA/アバター軽量化・Quest・iOS対応ツール から開きます。統合ウィンドウが開き、上部のチップで対象を PC だけ・Quest だけ・両方に切り替えられます(既定は両方)。

元アバターは一切変更しません。実行すると、PC 用に「{名前}_Opt」、Quest 用に「{名前}_Quest」という複製がシーンとプレファブの両方に生成されます。アップロードするのはこの複製です。

AAO(Avatar Optimizer)との連携について

本ツールは anatawa12 氏の AAO: Avatar Optimizer と処理を分業する設計です。実行時は、生成した複製にのみ次の AAO コンポーネントを自動で追加・設定します(元アバターには一切追加しません)。

- Trace and Optimize … アバターを走査して自動でできる限りの最適化を行うコンポーネント(未使用ボーンの削減や自動マージなど)。追加の自動最適化のために付与します(下記の Merge Skinned Mesh / Remove Mesh By BlendShape は Trace and Optimize が無くてもビルド時にAAOが適用します)。
- Merge Skinned Mesh … 顔以外の複数の SkinnedMeshRenderer を1つの SkinnedMeshRenderer へ統合します(SkinnedMesh統合の実体)。統合時は BlendShape 名を自動変更して重複を避けます。レンダラーをグループ1〜8に振り分けて、まとまりごとに統合するグループ指定モードもあります(顔は常に自動保護)。
- Remove Mesh By BlendShape … 指定した BlendShape で動く頂点とポリゴンを削除します(服の下に隠れる肌などを消す、隠れメッシュ削減の実体)。

PhysBone のマージ(設定が一致する兄弟チェーンを1本へまとめてコンポーネント数を減らす処理)は本ツール独自の実装で、AAO の Merge PhysBone は使用していません。

適用タイミング: これらの統合・削除は、アップロード/Play時のビルド(NDMF)で実行されます。そのため、生成直後の複製やエディタ上の診断値には反映されず、実際のアップロード後にさらに軽くなります。最終的な数値は VRChat SDK のビルド結果、または本ツールの実測レポート(後述)で確認してください。

AAO未導入時の挙動: AAO が無くても本ツール自体は動作します。AAO を使う機能(SkinnedMesh統合・隠れメッシュ削減など)は自動でスキップされ、ウィンドウ内に導入の案内が表示されます。導入は次のリポジトリを VCC / ALCOM に追加して行います。導入リポジトリ https://vpm.anatawa12.com/add-repo 、公式ドキュメント https://vpm.anatawa12.com/avatar-optimizer/ja/ 。

NDMFコンソールのAAO警告・最適化ログについて

変換直後の即時実測(1.4.0 以降)や ▶️(Play)・SDK ビルドを行うと、NDMF のコンソールに AAO(Avatar Optimizer)発の警告が出ることがあります。SkinnedMesh統合(Merge Skinned Mesh)まわりで代表的なのは、統合先(統合ターゲット)に Root Bone が設定されていない・統合先に Anchor Override(アンカー上書き)が設定されていない・統合ソース間で同名の BlendShape が違う値を持つ(値不一致)の3種です。いずれも 1.4.0 以前からビルド時には発生していたもので、即時実測で毎回コンソールを通るようになって見えやすくなっただけで、新しい不具合ではありません。

1.4.1 では、このうち機械的に解消できるものを変換側で自動処理します。統合先(Merge Skinned Mesh のターゲット)へ Root Bone と Anchor Override を自動設定して統合ソースの基準へ揃え、食い違い警告を抑えます。また、アニメーションで使われていない同名 BlendShape の値衝突は、現在の値でメッシュへ焼き込んで自動固定します(現在値のまま焼くので見た目は変わりません)。アニメーションで動かしている BlendShape は勝手に固定すると表情などが壊れるため自動固定せず、変換レポートで該当シェイプを案内します(必要なら手動で調整してください)。

1.7.1 では、統合そのものが逆効果になるレンダラーを自動で統合対象から外すガードを2つ追加しました。(1) 多重描画(マテリアルスロット数>サブメッシュ数。例: 髪の mat_hair + FakeShadow を1サブメッシュへ重ねているもの)は、AAO がビルド時に多重描画を実体化(サブメッシュを複製)してポリゴン数が増えるため統合しません(AAO 警告「複数マテリアルによるマルチパスレンダリング…パフォーマンスランクにおけるポリゴン数が増えるなどの影響」。実例では PC の _Opt 複製がビルド時に 68,674→78,948 三角形へ増え、PCの 70,000 の崖を越えて即 Very Poor になっていました)。(2) 統合対象のメッシュ間で material.(マテリアルプロパティ)アニメーションの対象が食い違う場合、そのアニメが向いているメッシュは統合しません(統合すると AAO がそのアニメを統合先メッシュ全体へ波及させ、意図しない部分にも適用されるため。AAO 警告「_UseMain2ndTex, _UseEmission へのアニメーションが異なります…意図していない部分にアニメーションが適用されてしまう可能性」。統合ソース全員が同じ material. アニメ集合ならそのまま統合します)。いずれも統合プレビューに理由(「多重描画…のため統合しません」「マテリアルアニメーションが統合先全体へ波及するため統合しません」)を表示します。あわせて、上記の同名 BlendShape の値不一致で「アニメーションで動かしているシェイプ」(例: ジャケットのトグルで動く shrink_for_jacket)は、実行時にアニメーションが値を決めるため統合対象間で現在値が食い違っていても多くの場合は無害で、変換レポートの案内にもその旨を明記しました(見た目が意図と異なるときだけ該当メッシュを統合から除外してください)。

1.9.0 では、1.7.1 で一律に統合対象外にしていた多重描画(マテリアルスロット数>サブメッシュ数)のレンダラーを、上描き(余剰)スロットの削除で統合できるようにしました。① 何も描かない上描きは自動削除して統合します。余剰スロットのうち、スロットが空、または Quest 対応で非表示化される疑似影 FakeShadow / アウトラインのみなど何も描かないものは、統合の直前に複製から自動で削除します。多重描画でなくなるため、ポリゴンを増やさず・見た目も変えずに通常どおり統合されます(Quest 対応では疑似影付きの髪などが操作なし・見た目の変化なしで自動統合され、統合後の概算が 3 に留まっていたものが 2 になります)。② 見える上描き(前髪の影など、実際に重ねて描かれている余剰スロット。例: PC の FakeShadow)は、統合プレビュー各行の「上描きスロットを削除して統合(前髪影などの重ね描き効果が消えます)」チェック(既定オフ)で削除して統合できます。重ね描き効果は失われますが、SkinnedMesh 数・スロット数を確実に減らせます(チェックすると理由が「上描きスロット削除により統合します」に変わります)。③ マテリアル差し替えアニメ(m_Materials.Array.data[N])の対象レンダラーは、削除するとスロット番号がずれてアニメが壊れるため、自動削除・チェックのいずれの対象にもしません(「マテリアル差し替えアニメがあるためスロット削除不可」と表示)。削除するのは常に余剰スロット(サブメッシュ数を超える分)だけで、基底のサブメッシュスロットには手を付けません。処理はすべて複製に対してのみ行い、元アバターは変更しません。PC軽量化・Quest対応の両パイプラインで有効です(PC は非表示変換が無いため、PC を対象に含むあいだは疑似影などの見える上描きは②のチェックで削除します)。

1.10.0 では、1.7.1 の**マテリアルアニメーションの波及ガードで統合対象から外されたメッシュを、そのマテリアルアニメーションを無効化することで統合できる**ようにしました。エミッション切り替え(_UseEmission / _UseMain2ndTex 等)などのギミックを持つシャツ・ネクタイのようなメッシュは、統合対象間で material.(マテリアルプロパティ)アニメの対象が食い違うため 1.7.1 のガードで自動除外され(「マテリアルアニメーションが統合先全体へ波及するため統合しません」)、SkinnedMesh 数が減らないことがありました。1.10.0 からは、ガードで除外された各行に「マテリアルアニメーションを無効化して統合(この切り替え演出は動かなくなります)」チェック(既定オフ)を表示します。オンにすると、そのレンダラーに向いた material.* アニメを**複製側のアニメーションクリップからのみ無効化(除去)**した上で統合します。無効化により波及の懸念がなくなるため統合先へまとまり、SkinnedMesh 数が減ります(例: 顔以外がシャツ・ネクタイのエミッション演出で分かれて統合後 4 になっていたものを、両方チェックして Quest の Poor 上限 SkinnedMesh 2 に収められます)。チェックした行は理由が「マテリアルアニメーションを無効化して統合します」に変わり、統合後の概算へ反映されます。無効化は複製に対してのみ行い、**元アバターやそれと共有するアニメーションクリップ・コントローラーは一切書き換えません**(トグル整理の m_IsActive 無効化と同じ、クローン専用の複製・差し替えの仕組みを共用します)。チェックした切り替え演出は動かなくなります(固定表示になります)。変換レポートに「'X': マテリアルアニメーションを無効化して統合しました(無効化: _UseEmission ほか n件 / クリップ: A, B)。該当の切り替え演出は動かなくなります。」を出します。マテリアル差し替えアニメ(m_Materials.Array.data[N])は従来どおり対象外(チェックは出ません)で、無効化できるのは material.(プロパティ値)のアニメだけです。PC軽量化・Quest対応の両パイプラインで有効です。

なお、AAO はビルド時に最適化の成果(削減したポリゴン数・統合したメッシュ数など)をコンソールへ出力します。行頭に「!」アイコンが付いて表示されることがありますが、これはエラーではなく AAO による正常な成果報告です。赤字のエラーでなければ変換は成功していますので、そのまま進めて問題ありません。

メニュー構成

RARA メニューの主な項目は次のとおりです。

- RARA/アバター軽量化・Quest・iOS対応ツール … 統合ウィンドウを開く(上部のチップで対象 PC / Quest / 両方を切り替え。既定は両方)
- RARA/実測レポート … 実測レポートウィンドウを開く(後述)

旧2ツールとの関係

このツールは、旧 PC軽量化ツール / Quest対応コンバーター と同じ変換エンジンを共有しています。中身の処理(トグル整理・SkinnedMesh統合・テクスチャ縮小・アトラス統合・PhysBone整理・シェーダー変換・ポリゴン削減など)は旧ツールと同一です。旧ウィンドウ(単体UI)はメニューからは外しましたが、クラスと Open() API はコード互換のために残しています。

このウィンドウに移植された Quest 向けパネル

旧 Quest対応コンバーターにあった Quest 固有の設定パネルを統合ウィンドウへ移植しました。

- Quest 除外(ビルド除外パス)… 構成整理ステップ
- 隠面メッシュ削減(服の下に隠れる肌などをブレンドシェイプで削除)… 構成整理ステップ
- アトラス統合 … マテリアルステップ
- 表情デカール(チーク/涙/アイハイライト)の再現・非表示 … マテリアルステップ
- Quest 変換設定(シェーダー・テクスチャ形式など)… マテリアルステップの末尾(実行の直前に最終確認)

使い方

上から順に進めれば設定できます。

1. アバターを選ぶ。シーンに置いたアバター(VRCAvatarDescriptor)を指定します。
2. 対象を選ぶ。PC / Quest のチップで切り替えます。既定は両方です。
3. プリセットを選ぶ(任意)。おまかせQuest対応・PCを最低Poorまで軽量化(Very Poor回避)・PCをGoodまで軽量化・フル両対応(PC+Quest)から、目的に近いものを押すと対象と目標ランクがまとめて設定されます。あとから各項目で微調整できます。
4. ① 診断。現在の値を PC 基準と Quest 基準の二列で確認し、対象ごとの目標ランクを選びます。PC のポリゴンは Excellent のみ 32,000・Good/Medium/Poor はすべて 70,000 で、70,000 を超えると一気に Very Poor になります(32,001〜70,000 は Good)。
5. ② 構成整理。トグルの固定、SkinnedMesh の統合を設定します。各トグルグループには「非表示固定で減る専有アセット」の削減量チップ(圧縮後 -◯MB / 展開後 -◯MB)が表示されます。専有アセットはそのグループのサブツリー内だけが使うテクスチャ・メッシュで、共有アセットしか持たないグループは単独では減りません(「共有テクスチャのみ」と表示)。サイズ超過時は削除候補を大きい順に提示するガイドも出ます(合計は親子の入れ子グループを二重計上しません)。SkinnedMesh 統合は「統合しない / 顔以外を統合(推奨) / グループ指定で統合」の3モードから選べます。グループ指定では各レンダラーをグループ1〜8に振り分け、まとまりごとに1つのメッシュへ統合します(顔は自動保護され、グループへ割り当てできません。未割り当てのレンダラーは統合されません)。Quest 対象のときは、続けて Quest 除外(ビルド除外)と隠面メッシュ削減もここで設定できます。
6. ③ PhysBone。残す・削除・マージを設定します(各 PhysBone の「残す」チェックは既定オン(チェック=残す)で、外すと削除。マージのオン/オフもこのセクション内で切り替えます)。選択後のコンポーネント数がPC/Questの上限とあわせて色付きで表示され、ランク別のコンポーネント上限ラダー(PC: Excellent 4 / Good 8 / Medium 16 / Poor 32、Quest: Excellent 0 / Good 4 / Medium 6 / Poor 8)と、次のランクまであと何個削ればよいかの目安、対象ごとの上限の要点行(PCは目標/Poor上限と超過時Very Poor(PhysBone・コンタクト・コンストレイントが全停止し得ます)、QuestはPoor上限超過で分野単位に全停止)も併記されます。超過している場合は「優先度で自動選択」を押すと、髪・胸などを優先して自動的に上限内へ収められます(絞り込みの基準は、PC単独なら目標PCランクのコンポーネント上限、Quest対象ならQuest上限8本。ボタン名と説明にどちらの基準かを表示します)。
7. ④ テクスチャ。縮小するテクスチャを選びます。サイズ診断は圧縮後(ダウンロード)10MBと展開後(非圧縮メモリ)40MBを独立した上限として並べて判定します。「上限内まで自動調整(圧縮後10MB・展開後40MB)」で両上限を同時に狙う縮小計画を自動作成できます(テクスチャを最小まで下げても展開後がメッシュ・ブレンドシェイプ支配で届かない場合は、その旨を正直に表示し、Quest除外・メッシュ削減の併用を案内します)。
8. ⑤ ポリゴン。Quest 向けのポリゴン削減(Meshia連携)を設定します。目標三角形数を指定し、ワンボタンで「_Quest」複製へ Meshia Mesh Simplification を追加すると、ビルド時に高品質な簡略化が適用されます(自前の簡略化 UI は 1.6.0 で廃止)。削減結果はエディタ診断には出ないため、即時実測/実測レポートで確認します。Meshia 未導入時はパネルに導入手順が表示されます。旧バージョンで保存した自前のポリゴン削減計画は 1.6.0 では適用されません。
9. ⑥ マテリアル。PC 列と Quest 列を並べて、アトラス統合や変換方法を設定します。PC 列のアトラス統合では、lilToon のマットキャップ(映り込み)を持つマテリアルは統合時にマットキャップを外します(UV 再配置でマットキャップのマスクが壊れ、全面が白飛びするため)。映り込みを残したいマテリアルは「アトラス除外」に指定してください(統合対象から外れ、マットキャップも保たれます。外したマテリアルは変換レポートと統合プレビューに表示されます)。Quest 対象のときは、アトラス統合・表情デカール(チーク/涙/アイハイライト)・Quest 変換設定(シェーダー・テクスチャ形式など)もこのステップで設定します。Quest のアトラス統合では、既にQuest対応(VRChat/Mobile/Toon Standard・Toon Lit)のマテリアルも統合の対象になり(1.8.0)、購入したQuest対応済みアバターや元アバターを持たない過去の変換結果(RARA_Atlas_* を含む)でもスロットを削減できます(MatCap Lit・パーティクル系は対象外)。変換設定の詳細設定では、マットキャップの引き継ぎ(既定オン。オフ=削除相当。映り込みが乱れる場合はオフ推奨。Toon Standard 変換時のみ表示)を切り替えられます。Network ID の割り当て(PC/Quest間の揺れ物の掴み同期)は統合ウィンドウでは常にオンで、詳細設定に項目はありません(オン/オフを切り替えたい場合は単体の Quest 対応ツールの詳細設定・単体の PC 軽量化ツールの実行セクションを使います)。変換設定は実行の直前に最終確認できるよう末尾に配置しています。
10. ⑦ 実行。実行オプション(プレファブ保存・AAO付与)を確認し、「この実行で行われること」の要約と実行前の確認ダイアログの内容を確認して実行してください。実行後は、変換前後の比較テーブル(PC/Quest、目標との突き合わせ)と、Quest の変換結果チェック(元 ← → 変換後マテリアルのサムネイル比較)が結果欄に表示されます。
11. ⑧ ヘルプ / はじめての方へ。かんたんな説明(3行)・軽くなる仕組み(処理→何が減るか)・Quest/iOS版の割り切り・迷ったら3ステップ・4段階ランクの主要しきい値の目安をまとめています。はじめての方はここから読むと迷いにくくなります。

診断の目標達成ガイド

① 診断の下部に「目標達成ガイド」が出ます。選んだ目標ランクを超えている項目があると、その項目ごとに推奨アクションと「→ ステップnで対応」ボタンが並びます。ボタンを押すと、対応するステップ(テクスチャ・ポリゴン・PhysBone など)が開きます。

繰り返し再生成の安全性

このツールは、生成した結果を見て設定を直し、また生成する、という繰り返しを前提に作られています。

元プレファブは非破壊です。生成物は元アバターとは別の複製です。
何度でも再生成できます。実行のたびに前回の「_Opt」「_Quest」複製は自動で作り直されるため、複製が増え続けることはありません。
古い複製は自動で削除して作り直します。生成アセットは決まった場所へ上書き保存されるので、参照が壊れることもありません。
元アバターの名前を変えたあとに取り残された古い複製がある場合は、実行結果に名前付きで警告を出します(自動では削除しません)。

導入条件

AvatarOptimizer(AAO)の事前導入を強く推奨します。メッシュ・マテリアルスロットの統合や非表示メッシュの削減はビルド時に AAO が担当する分業設計のため、AAO が無いと削減効果が出にくく、Quest ではスキンメッシュ数の上限に収まらず Very Poor 判定になりやすく、既定で相手にフォールバック表示され揺れ物も停止し得ます(アップロード可否自体はサイズ上限のみで決まります)。導入は VCC/ALCOM にリポジトリを追加して行います。ブラウザで https://vpm.anatawa12.com/add-repo を開き、VCC/ALCOM が開いたら Add Repository → プロジェクトに AvatarOptimizer を追加 → Unity へ戻ってください。

注意

AAO(AvatarOptimizer の Trace and Optimize)によるメッシュやマテリアルスロットの統合、非表示メッシュの削減は、ビルド(アップロード)時に反映されます。そのため、生成直後にシーン上や診断表で見える数値は、最終的なビルド後よりも控えめ(改善が少なめ)に出ます。実際のアップロード後はさらに軽くなる場合があります。ビルド後の本当の数値は、下記の実測レポートで確認できます(_Opt / _Quest 複製を ▶️(Play)するか、SDK で Build & Test / Upload すると、最終的な複製そのものを実測します)。

1.5.1 から、EditorOnly タグ付き・Quest除外(Quest 対象時)に登録したオブジェクトは、診断だけでなく各パネルの一覧・プレビュー(マテリアル・衣装/トグル整理・SkinnedMesh統合・PhysBone など)と変換からも自動で外れます(ビルドに含まれないため。統合前後の概算にも数えません)。非表示にした項目があるパネルには「EditorOnly/Quest除外のため n 件を非表示(ビルドに含まれないため)」を1行だけ表示します。Quest除外の登録パネルだけは全件を表示し、既に EditorOnly タグが付いているものには「EditorOnly」バッジを付けます。

実測レポート(変換直後の自動実測・ビルド/Play 時の実測)

「見積り」ではなく、ビルド/Play 時の最終的な複製そのものを VRChat SDK と同じ計算で実測して確認できる機能です。メニューの RARA/実測レポート から開きます。MA(Modular Avatar)・AAO・Meshia・lilToon などの適用後、EditorOnly 除去後の姿を測るため、上記の「エディタ数値は控えめ」の注意をそのまま実測で確かめられます。1.4.0 からは、変換が完了すると自動でビルド不要の即時実測が走り、生成直後の複製の最終値を ▶️(Play)や SDK ビルドを待たずにその場で確認できます(下記)。

- 変換直後の自動実測(ビルド不要): 変換が完了すると、自動で「ビルド不要の実測」が走ります。NDMF の手動ベイク API を使って、ビルド時と同じ MA(Modular Avatar)・AAO などの処理を一時コピーへ適用し、数秒で測定します(測定に使った一時コピーと生成した中間アセットは自動で削除されます)。▶️(Play)や SDK ビルドを待たずに、生成直後の複製の最終的なランク・項目値をその場で確認できます。実測レポートウィンドウの「今すぐ実測(ビルド不要)」ボタンからも、いつでも同じ即時実測を実行できます。自動実測は設定でオフにできます(既定オン)。
- 即時実測の範囲(正直な注記): ランク・ポリゴン数などの計算は、ビルド/Play 時の実測とまったく同じ結果になります。ただしダウンロードサイズ(10MB 判定)の実ファイルサイズは、本物の SDK ビルド/アップロードでしか得られないため、即時実測では推定表示のままです。また lilToon など SDK ビルド固有の処理は即時実測には含まれません(この点は ▶️(Play)・SDK ビルド時の実測のほうがより厳密です)。NDMF が未導入のときは即時実測をスキップします。
- ▶️(Play)で実測: _Opt / _Quest 複製を ▶️(Play)すると、ビルド時と同じ前処理を通した最終複製を実測し、結果がポップアップ表示されます。PC 基準・Quest 基準の両方の総合ランク、項目別の値(ダウンロードサイズ・三角数・スキンメッシュ数・マテリアルスロット数・テクスチャメモリ・PhysBone 各種など)、目標ランクとの達成判定、元アバターとの差を並べます。
- 実バンドルサイズの記録: VRChat SDK の Build & Test / Upload でビルドすると、生成されたアセットバンドルの実ファイルサイズも記録されます(エディタの推定ではなく、実際にアップロードされるバンドルサイズ)。Quest/Android では 10MB 上限との判定を添えて表示します。
- 対象範囲: 既定では名前が _Opt / _Quest で終わる複製だけを計測します。「_Opt / _Quest 以外のアバターも計測する」をオンにすると全アバターを対象にできます。編集モードでは「再計測」でスタジオ診断による測り直しもできます。
- オフにする方法: 計測自体を止めたいときは、実測レポートウィンドウ上部の「ビルド/Play時に実測する」をオフにします(ビルドや Play の動作自体には影響しません)。Play/ビルドのたびの自動表示だけを止めたいときは「▶️(Play)・ビルドのたびに実測レポートを表示する」をオフにします。

Meshia(ポリゴン削減の実体)について

Meshia Mesh Simplification(Ram.Type-0 氏 / MIT)は、NDMF でビルド時(Play / アップロード)にメッシュを高品質に簡略化するツールです。1.6.0 から、本ツールの⑤ ポリゴン削減はこの Meshia へ連携する方式になりました。目標三角形数を指定すると、本ツールが「_Quest」複製の子オブジェクトへ Meshia のアバター単位コンポーネント(Meshia Cascading Avatar Mesh Simplifier)をワンボタンで追加・設定します(この自動付与には Modular Avatar が必要です。未導入時はパネルに導入案内が出るだけで付与されません)。導入は VPM リポジトリ https://ramtype0.github.io/VpmRepository/ を VCC / ALCOM に追加して行います(公式ドキュメント https://ramtype0.github.io/Meshia.MeshSimplification/ )。

使い方のパターン。

- ワンボタン運用(推奨): 目標三角形数を指定して「_Quest 複製へ Meshia を追加」を押すだけ。顔・髪を含む配分は本ツールが目標へ合わせて設定します。追加後に Meshia コンポーネント側で個別の微調整もできます。
- レンダラー単位で追い込む: 崩れやすい/見せ場の部位は、追加された Meshia Cascading Avatar Mesh Simplifier で部位ごとに目標三角形数や保護設定を調整します(さらに細かく詰めたいときは、レンダラー単位の Meshia Mesh Simplifier を手動で足す方法もあります)。EditMode プレビューで品質を確認できます。
- 二重指定に注意: 同一レンダラーに、本ツールが追加した Meshia と手動で足した Meshia の両方を効かせないでください(レンダラー単位でどちらか一方にする)。両方が同じ目標へ強くかかると、ビルド後に削りすぎになります。

注意(ビルド時反映): Meshia の削減はビルド時に反映されるため、エディタ/診断のポリゴン数には出ません(EditMode プレビューはありますが、実メッシュの差し替えはビルド時です)。最終的な三角形数は即時実測(1.4.0 以降)または実測レポート(▶️ Play / SDK ビルド)で確認してください。⑤ ポリゴン削減パネルは、手動で追加された Meshia コンポーネントを検出すると二重指定の注意を表示します。旧バージョンで保存した自前のポリゴン削減計画は 1.6.0 では適用されません。

謝辞: Meshia Mesh Simplification(MIT License / Ram.Type-0 氏)に感謝します。

対応元シェーダー(変換元)

Quest / iOS 変換では、変換元(PC 側)のマテリアルが次のシェーダーファミリーなら質感を読み取って Toon Standard / Toon Lit へ自動変換します。lilToon が最も忠実に再現でき、それ以外のファミリーは「プロパティ読み取り+メインテクスチャへのティント焼き込み」による近似変換です(色調補正は結果色をテクスチャへ焼き込んで近づけるため、PC 版と完全一致はしません)。1.3.0 で Unity Standard/Unlit・VRM MToon(VRoid)を変換元として追加し、Poiyomi Toon はロック済みマテリアルまで対応しました。

- lilToon … 最も忠実。色調補正・影(2nd/3rd)・エミッション・マットキャップ・最低明るさ・カリングをベイクして引き継ぎます。
- NonToon(jp.lilxyzw)… ベース色・ノーマル・影ランプ・マットキャップ(乗算/加算の2枠→1枠に近似)・カリングを引き継ぎます。エミッションは非搭載のため付きません。
- Poiyomi Toon(ロック済み含む)… メイン色(色相/彩度/明度/ガンマ)をティント焼き込みし、ノーマル・エミッション・カリングを引き継ぎます。影ランプは SDK 既定で、リムや独自の陰影は再現しません。ロック済み(最適化済み)も変換でき、焼き込みで失われたプロパティは既定値へ縮退します。
- Unity Standard / Unlit … メイン色・ノーマル・エミッション・カットアウト/透過方針・カリングを引き継ぎます。メタリック/スムースネスは無視するため、金属パーツは⑥ マテリアルの変換方法で MatCap Lit を選ぶと映り込みを付けられます。
- VRM MToon(VRoid)… メイン色・ノーマル・エミッション・カットアウト/透過方針・カリングを引き継ぎます。SphereAdd(スフィアマップ加算)は Toon Standard の内蔵マットキャップへ加算で引き継ぎます。

共通の方針として、カットアウト/透過の扱いは従来の透過ポリシー(半透明は乗算/加算で近似・顔/大型メッシュ/髪は保護して不透明化・カットアウトは塗りつぶし)に準拠し、ノーマルマップ・エミッション・マットキャップ・カリングはファミリー横断で引き継ぎます。1.4.2 からは顔を自動保護し、顔(アバターディスクリプタのビセーム/まばたきメッシュ、または名前に face・顔 等を含むメッシュ/マテリアル)の基底マテリアルは、透過設定でも半透明の自動再現(パーティクル化)や非表示の対象にせず常に不透明(Toon Standard)へ変換します(顔の白飛び・消失を防止)。チーク・涙・ハイライトなどのオプション重ねマテリアル(名前で判定)は従来どおり乗算/加算で近似します(既存の大型メッシュ・髪の保護と同じ扱い)。近似・非対応(いずれも変換は止めず変換レポートに警告)は次のとおりです。シェーダー固有の色調補正・デカール・ラメ(グリッター)などの特殊表現は再現されません。Unity Standard のメタリック/スムースネスは無視されます(金属感が要るパーツは MatCap Lit を推奨)。アウトラインは従来どおり Quest 非対応のため外れます。lilToon 以外は結果色をテクスチャへ焼き込む近似のため、ライティング応答や特殊効果は PC 版と一致しません。

マットキャップ(金属・ツヤの映り込み)

金属パーツやツヤのある面に使う「マットキャップ(MatCap)」を、Quest/iOS 変換でも近い見た目で再現します。⑥ マテリアルの Quest 列で設定します。

自動引き継ぎ(既定・Toon Standard)。lilToon のマットキャップは 1.1.0 から、変換先の Toon Standard が持つ内蔵マットキャップへ自動で引き継がれます(設定不要)。この引き継ぎは変換設定の詳細設定「マットキャップを引き継ぐ」でオン/オフでき(既定オン)、オフにすると引き継がれません(削除相当。映り込みが乱れる場合はオフを推奨します)。NonToon も従来どおり近似変換され、VRM MToon の加算スフィア(SphereAdd)も加算マットキャップとして引き継がれます。Poiyomi のマットキャップは自動では引き継がれません(変換レポートに表示。金属パーツはマテリアル別指定「MatCap Lit(金属向け)」でマットキャップ画像を引き継げます)。引き継がれるのはマットキャップ画像・合成モード(乗算はそのまま乗算、通常/加算/スクリーンは加算で近似)・強さ・マスクです。2ndマットキャップ(2枚目)や、カスタム法線・視差(パララックス)・カスタム UV などの高度なオプションは引き継がれません(変換レポートに警告が出ます)。

MatCap Lit(金属パーツ向けの変換先)。⑥ マテリアルの変換方法で、変換先に VRChat/Mobile/MatCap Lit を選べます。メインテクスチャ × マットキャップの乗算にライティングを足したシンプルなシェーダーで、金属のツルッとした映り込みに最適です。制約は、不透明のみ・ノーマルマップ/影ランプなし・マットキャップの性質上アトラス統合の対象外(マテリアルスロットを1つ使う)です。使い分けの目安として、通常は自動引き継ぎ(Toon Standard)で十分で、より強い金属感が欲しいパーツだけ MatCap Lit を選びます。

アトラス統合との関係。異なるマットキャップを持つマテリアル同士は、同じアトラスに統合されません(見た目を守るための仕様)。マットキャップを使うマテリアルはスロットを1つ使うことを見込んでおいてください。

見た目がおかしいときは(Quest/iOS)

Quest/iOS変換後に、見えるべきものが消えた、または消えるべきものが見えている場合は、⑥マテリアルのQuestパネルで該当マテリアルの変換方法を選び直して再生成してください(見せたい→Toon Standard(不透明)や乗算・加算、消したい→非表示)。非表示になった行には「不透明にする」「乗算で再現」のショートカットも用意しています。急ぎの場合は、生成された _Quest 複製のマテリアルのシェーダーを Inspector で直接 VRChat/Mobile/Toon Standard などの Quest対応シェーダーへ変更しても表示できます。ただし再生成すると生成マテリアルは同じ場所へ上書きされ、この手動変更は失われるため、恒久的な対応としてはパネルでの変換方法指定を推奨します。Quest上では VRChat/Mobile 系のシェーダーだけが表示され、それ以外はフォールバックします。アップロード前に Build & Test で実機相当の見え方を確認してください。

アップロード手順ガイド(PC/Quest 両対応)

⑪ 実行の末尾に、1つの Blueprint ID で PC(_Opt)と Quest(_Quest)を同一アバターの両対応版(フォールバック)として上げる4手順ガイドが常設されます。①ビルドターゲットを Windows に切替 → SDKパネルで _Opt をアップロード、②PC版の Blueprint ID を Quest版へ同期(別IDがある場合は確認ダイアログ)、③ビルドターゲットを Android に切替、④SDKパネルで _Quest をアップロード、の順です。現在のビルドターゲットと各複製の Blueprint ID を状態表示し、_Opt と _Quest が同一IDかどうかも色付きで示します。

1.5.1 から、手順①に「_Optだけをアクティブ化」、手順④に「_Questだけをアクティブ化」ボタンを追加しました。アップロードする複製だけをシーンでアクティブにし、元アバターともう一方の複製(_Opt / _Quest)だけを非アクティブにします(無関係なアバターや非アバターには触れません。Undoで元に戻せます)。SDKのBuilderパネルはアクティブなシーン内アバターを一覧するため、これで誤って別のアバター(PC用シェーダーのままの _Opt や元アバター)を選んでしまう事故を防げます。ボタンの隣に現在のアクティブ状態(_Opt / _Quest / 元)を表示します。あわせて、ビルドターゲットが Android / iOS なのに _Quest が非アクティブで元または _Opt がアクティブなときは赤い警告(SDKパネルでPC用アバターを選んでいると Mobile 非対応シェーダーの警告が大量に出てビルドできない旨)を、Windows なのに _Opt が非アクティブ・_Quest がアクティブなときは軽い情報表示を出します。

なお、SDKの「Multi-Platform Build & Publish」(複数プラットフォーム同時ビルド)は使わないでください。本ツールはPC版とQuest版を別々の複製でアップロードする方式のため、プラットフォームは1つずつ選び、手順①〜④の順に進めます。
