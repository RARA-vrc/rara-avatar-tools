# RARA Avatar Tools（アバター軽量化・Quest / iOS対応ツール）

VRChatアバターの **PC軽量化** と **Quest / iOS（Android）対応変換** を、1つの統合ウィンドウ（Avatar Studio）でまとめて行うUnityエディタ拡張です。元アバターは非破壊のまま、PC用『_Opt』とQuest用『_Quest』の複製を生成します。

[![導入ページ](https://img.shields.io/badge/VCC%20%2F%20ALCOM-listing-4c8bf5)](https://rara-vrc.github.io/rara-avatar-tools/) [![License: MIT](https://img.shields.io/badge/License-MIT-green)](./LICENSE)

> Unityメニューに追加される項目は **`RARA / アバター軽量化・Quest・iOS対応ツール`** の1つだけです。このウィンドウ内で対象（PC / Quest）を切り替えて使います。

---

## AAO（Avatar Optimizer）との連携について

本ツールは anatawa12 氏の AAO: Avatar Optimizer と処理を分業する設計です。実行時は、生成した複製にのみ次のAAOコンポーネントを自動で追加・設定します（元アバターには一切追加しません）。

- Trace and Optimize … アバターを走査して自動でできる限りの最適化を行うコンポーネント（未使用ボーンの削減や自動マージなど）。追加の自動最適化のために付与します（下記の Merge Skinned Mesh / Remove Mesh By BlendShape は Trace and Optimize が無くてもビルド時にAAOが適用します）。
- Merge Skinned Mesh … 顔以外の複数の SkinnedMeshRenderer を1つの SkinnedMeshRenderer へ統合します（「SkinnedMesh統合」の実体）。統合時は BlendShape 名を自動変更して重複を避けます。
- Remove Mesh By BlendShape … 指定した BlendShape で動く頂点とポリゴンを削除します（服の下に隠れる肌などを消す「隠れメッシュ削減」の実体）。

適用タイミング: これらの統合・削除は、アップロード/Play時のビルド（NDMF）で実行されます。そのため、生成直後の複製やエディタ上の診断値には反映されず、実際のアップロード後にさらに軽くなります。最終的な数値は VRChat SDK のビルド結果で確認してください。

AAO未導入時の挙動: AAO が無くても本ツール自体は動作します。AAO を使う機能（SkinnedMesh統合・隠れメッシュ削減など）は自動でスキップされ、ウィンドウ内に導入の案内が表示されます。導入は次のリポジトリを VCC / ALCOM に追加して行います。

- 導入リポジトリ: <https://vpm.anatawa12.com/add-repo>
- 公式ドキュメント: <https://vpm.anatawa12.com/avatar-optimizer/ja/>

謝辞: AAO: Avatar Optimizer（MIT License / anatawa12 氏）に感謝します。あわせて VRChat SDK・lilToon・NonToon・Poiyomi・Modular Avatar / NDMF にも感謝します。

## 1. これは何

VRChatアバターの重さ（パフォーマンスランク）を下げ、必要ならQuest / iOS版へ変換するためのツールです。対象ユーザーを次の3層に分けて設計しています。

- **とりあえずQuest / スマホ対応にしたい人** … シェーダーをMobile系へ変換し、質感をテクスチャへベイクして、Android / iOSで表示できる『_Quest』複製を作ります。
- **PCのまま Very Poor を避けたい人** … シェーダー変換はせず、テクスチャメモリ・マテリアルスロット・メッシュ・PhysBoneを削減して PC（Windows）基準でランクを Poor 以上へ引き上げます。
- **PCで Good / Excellent を狙いたい人** … 同じくPC基準で、より上のランクへ寄せます。

すべての操作は1つのウィンドウで完結します。内部的には **PC最適化エンジン（PC Optimizer）** と **Quest / iOS変換エンジン（Quest Converter）** を共有しています。

### 対応シェーダー

| シェーダー系統 | Quest / iOS変換 | 備考 |
|---|---|---|
| **lilToon** | ✅ | 色調補正・影・2nd/3rdレイヤー・エミッションをベイクして Toon Standard / Toon Lit へ変換。ベイクにlilToon付属のベイク用シェーダーを使用 |
| **NonToon**（jp.lilxyzw） | ✅ | ベーステクスチャ・ノーマルを引き継ぎ、影グラデーションを影ランプへ生成、マットキャップは近似。エミッションは非搭載のため付かない |
| **Poiyomi**（Poiyomi Toon） | ✅ | メイン色調補正をベイク、ノーマル・エミッションを引き継いで Toon Standard へ。**ロック済み（最適化済み）マテリアルも変換可** |
| **Standard系** | ✅ | Toon Standard / Toon Lit へ変換 |

- **FakeShadow（前髪影）・OutlineOnly などの効果専用シェーダー** は、Questでは白い不透明面としてベイクされてしまうため **自動で非表示化** されます（ベース面はそのまま変換）。
- PC軽量化のみを行う場合、PCではシェーダーを変換しないため、元マテリアルのシェーダーがそのまま使われます（lilToonは必須ではありません）。

### 動作環境

- **Unity 2022.3系**（VRChat公式指定バージョン）
- **VRChat Avatars SDK `>=3.8.1`**（`com.vrchat.avatars`。変換先の VRChat/Mobile/Toon Standard シェーダーがこのバージョンで追加されたため）
- **レンダーパイプラインは Built-in のみ対応**
- **AvatarOptimizer（AAO）を強く推奨**
  メッシュ・マテリアルスロットの統合や隠れメッシュ削減は **ビルド時にAAOが担当する分業設計** です。AAOが無いと削減効果が出にくく、特にQuestではスキンメッシュ数の上限に収まらず Very Poor になりやすく、既定で相手にフォールバック表示され揺れ物も停止し得ます（アップロード可否自体はサイズ上限のみで決まります。実質必須）。
  導入用リポジトリ: <https://vpm.anatawa12.com/add-repo>
- **Modular Avatar / NDMF は任意**（導入されていれば MergeAnimator・Material Setter・メニュー切替なども変換対象として扱います）

## 2. インストール（VCC / ALCOM）

VPMパッケージ `jp.rara.avatar-tools` として導入します。

### ワンクリックで追加

導入ページを開いて「VCC / ALCOM に追加」ボタンを押すか、次のリンクをブラウザで開きます（VCC / ALCOM が起動し、リポジトリ追加の確認画面が開きます）。

- 導入ページ: <https://rara-vrc.github.io/rara-avatar-tools/>
- ワンクリック: `vcc://vpm/addRepo?url=https%3A%2F%2Frara-vrc.github.io%2Frara-avatar-tools%2Findex.json`

### 手動でリポジトリを追加

VCC / ALCOM の **Settings > Packages > Add Repository** に、次のURLを貼り付けます。

```
https://rara-vrc.github.io/rara-avatar-tools/index.json
```

### プロジェクトへの追加

1. 対象のアバタープロジェクトを VCC / ALCOM で開きます。
2. **Manage Project** の一覧から **RARA Avatar Tools** を追加します。
3. Unityへ戻ると、コンパイル後にメニューバーへ `RARA / アバター軽量化・Quest・iOS対応ツール` が追加されます。

追加の設定は不要です。アンインストールは Manage Project からパッケージを外すだけです。

### 旧 unitypackage版からの自動移行

以前 `Assets/RARA/` 以下へ unitypackage で導入していた場合、VPMパッケージが持つ **legacyFolders** の定義により、パッケージ導入時に旧フォルダ（`Assets/RARA/QuestConverter/Editor`・`Shaders`、`Assets/RARA/PCOptimizer/Editor`、`Assets/RARA/AvatarStudio/Editor`）が自動で除去され、`Packages/jp.rara.avatar-tools/` 側へ一本化されます。旧版と新版のスクリプトが二重に存在してコンパイルが衝突することはありません。

## 3. クイックスタート（3手順）

1. **プリセットを押す** — シーンのアバター（`VRCAvatarDescriptor`）を選び、目的に近いプリセット（Questだけなら「雑にQuest対応」、両方なら「フル両対応（おすすめ）」）を押します。対象と目標ランクがまとめて設定されます。
2. **⑦ 実行** — 「実行（選択した対象へ生成）」を押して複製を生成します。元アバターは変更されません。
3. **Build & Test** — 生成された複製を VRChat SDK コントロールパネルでアップロード（またはBuild & Test）します。最終的なランク・見た目はSDKで確認します。

---

## 4. ウィンドウ完全ガイド

ウィンドウは上から順に進めば完了する構成です。ヘッダー → アバター選択 → 要約行 → 対象チップ → プリセット → ①〜⑧ステップ、の順で並びます。

### ヘッダー / アバター選択 / 要約行

- **アバター**: シーン上の `VRCAvatarDescriptor` を指定します。「シーンから検出」で自動選択もできます（`_Opt` / `_Quest` 複製は避けて元アバターを優先）。
- **要約行**: 選択アバター名、PC総合ランク、Quest総合ランク、Quest推定ダウンロードサイズを常時表示します（診断キャッシュの読み取りのみで再計測しません）。

### 対象チップ（PC / Quest・既定で両方）

`PC` と `Quest / iOS` のチップで対象を切り替えます。**既定は両方**です。対象を変えると診断はリセットされ、次に診断を開いたとき再計測されます。どちらも未選択だと警告が出て実行できません。

### プリセット（4種）

| プリセット | 対象 | 目標ランク | 補足 |
|---|---|---|---|
| **雑にQuest対応** | Questのみ | Poor | 透過は乗算/加算で自動再現（ストッキング等の透けを保持）。ポリゴン削減は既定オフ |
| **PCをPoorに** | PCのみ | Poor | |
| **PCをGoodに** | PCのみ | Good | |
| **フル両対応（おすすめ）** | PC + Quest | PC=Good / Quest=Poor | 透過は自動再現・ポリゴン削減は既定オフ |

押した後も各ステップで個別に微調整できます。

---

### ① 診断（PC / Quest 現在値と目標）

現在値をPC基準とQuest基準の **2列で並記** します。

- **目標ランクピッカー**: PC・Questそれぞれに `Excellent / Good / Medium / Poor` のトグルがあり、超過項目を強調表示します。
- **総合ランク要約**: PC総合・Quest総合を色付きで表示。ダウンロードサイズが10MB上限を超える見込みのときだけ「Androidアップロード不可（サイズ上限超過）」を明示し、Very Poor は既定でフォールバック表示になる旨を注意表示します。
- **比較テーブル**: 項目ごとにPC現在値・PCランク・Quest現在値・Questランクを並べ、目標を超えた項目を赤字＋⚠で示します。
- **Quest詳細**: 推定ダウンロードサイズ（10MB上限との比較）、Quest非対応シェーダーのマテリアル一覧、Android非対応コンポーネント一覧、テクスチャ設定の警告を表示します。
- **目標達成ガイド**: 目標を超えた項目ごとに推奨アクションと「→ ステップnで対応」ボタンが並び、押すと対応ステップ（テクスチャ・ポリゴン・PhysBone・構成整理・マテリアル等）が開きます。

### ② 構成整理（トグル / SkinnedMesh統合）

トグルの固定とSkinnedMeshの統合を設定します。どちらも **統合の実体はビルド時にAAOが実施** するため、表示される数値は概算です。

**衣装・トグル整理**（トグル固定の2経路）
- 各グループを `維持 / 常時表示 / 非表示除去` から選びます。
  - **常時表示** = 常時ON固定にしてトグルアニメを外す → AAOがビルド時に同一メッシュへ結合でき、SkinnedMesh数・スロット数が減る。
  - **非表示除去** = そのメッシュをEditorOnly化して除去（AAO不要）。
- 一括ボタン「**現在の表示状態で固定（全トグル）**」「**すべて維持に戻す**」、各グループの「ピン」でシーン上をハイライトできます。固定するトグル数（表示固定/非表示固定の内訳）を予測表示します。

**SkinnedMesh統合（顔以外を1つへ）**
- `統合しない / 顔以外を統合（推奨）` を選びます。顔（ビセーム / まばたきメッシュ）以外の SkinnedMeshRenderer を1つへ統合し、SkinnedMesh数・スロット数を削減します（Questのスキンメッシュ上限2対策）。
- 顔メッシュ・Clothの付いたメッシュ・EditorOnly・プレビューで「統合しない」を選んだメッシュは対象外です。統合前→統合後（概算）のSMR数と、レンダラーごとの「統合する/しない＋理由」を表示します。
- **注意（トグル無効化）**: 統合先は常時表示になるため、統合したメッシュのON/OFFトグルは効かなくなります（切り替えを残したいメッシュは「統合しない」を選ぶか、先に「常時表示」で固定してから統合します）。表情・ブレンドシェイプアニメは、AAOがブレンドシェイプ改名・スロット再マップ・アニメ再パスを自動で行うため追従します。

**Quest固有（対象がQuestのとき）**
- **Quest除外（ビルド除外パス）**: ドラッグ&ドロップで登録したオブジェクトを、Quest複製でのみ EditorOnly化してビルドから完全除去します（PC版には影響しません）。解決チェック・ピン・削除つき。アバタールートやスキンメッシュが参照するボーンは登録できません（同名兄弟・名前に「/」を含むものも検証で弾かれます）。
- **隠面メッシュ削減**: 服の下に隠れて見えない肌などを、アバターが持つ shrink（縮小）ブレンドシェイプで削除します（AAO必須。未導入時は導入CTAを表示）。候補ごとにチェック・使用ブレンドシェイプ名・理由・ピンを表示。shrinkブレンドシェイプが無い場合は AAO の Remove Mesh in Box / By Mask / By UV Tile による手動手引きを表示します。

### ③ PhysBone 整理

残す・削除・マージを設定してコンポーネント数を削減します（揺れは維持）。

- **マージ設定**: `設定一致チェーンをマージ` と `設定違いも先頭へ統一（ルーズ）` の2種。ルーズは設定が完全一致しない兄弟チェーンも先頭の設定に統一します（アニメ制御・パラメータ使用などの安全ゲートは維持）。
- **選択後コンポーネント数** を、対象（PC / Quest）ごとの上限メーター・**ランク別コンポーネント上限ラダー**・次ランクまでの削減目安と併せて色付き表示します。
  - PC上限ラダー: **Excellent 4 / Good 8 / Medium 16 / Poor 32**
  - Quest上限ラダー: **Excellent 0 / Good 4 / Medium 6 / Poor 8**
- **優先度で自動選択（Quest 8本まで）**: 髪・胸などの名前パターンで優先度を判定し、Quest上限（8本）内に収まるよう自動でPhysBoneを残し、残りを「削除」に設定します。**Questで8本を超えるとモバイルで全ての揺れが停止するため**、超過時は赤の強警告を表示します。
- **行の操作**: グループ行は「マージ n本 → 1本」表示＋「**マージしない**」（このグループを1本へまとめず個別に残す）＋「削除」。単独行は「削除」。削除したPhysBoneは別枠の「削除指定」一覧に移り、「**戻す**」で復元できます（見つからないパスは黄色で表示）。
- PCは上限を超えてもアップロード可能（ランクが下がるだけ）です。

### ④ テクスチャ縮小

縮小しても **元テクスチャとそのインポート設定は一切変更せず**、生成時に低解像度コピーを作って複製がそれを参照します。

**PC（VRAM）**
- 現在のPCテクスチャメモリを目標ランク上限と比較表示。寄与の大きいものから縮小提案が並びます。
- 「**全て縮小計画へ追加**」「**目標まで自動調整**」（目標MBを下回るまで効果の大きい順に登録）「**縮小計画をクリア**」、各行の「追加」で個別登録できます。

**Quest（ダウンロードサイズ）**
- 推定ダウンロードサイズを10MB上限と比較（テクスチャ / メッシュ / アニメの内訳つき）。
- 「**10MB以下まで自動調整**」（確認ダイアログで内容を見てから一括登録・5%マージン）「**縮小計画をクリア**」。ダウンロードサイズの大きいテクスチャ上位を一覧し、Android向けインポート上書きが無いものには「Android上書きなし」バッジを付けます。縮小・削減提案から個別に登録もできます。

---

### ⑤ ポリゴン削減（Quest）

Quest対象のときに設定できます。**既定は無効**（明示的に有効化）。

- 「**ポリゴン削減を有効化**」トグル。無効のまま現在の三角形数が目標を超えていると、琥珀の案内と「**ポリゴン削減を有効にする**」ボタンが出ます。
- **目標ランク**を `Excellent 7,500 / Good 10,000 / Medium 15,000 / Poor 20,000` から選ぶか、目標三角形数を直接入力します。「**自動で配分計画を作成**」でメッシュごとに削減目標を配分します（優先度の低い その他 → 衣装 → 素体 → 髪 → 顔 の順に削る）。「計画クリア」あり。
- 各行は **カテゴリバッジ（顔 / 髪 / 素体 / 衣装 / その他）** ＋現在→目標（編集可能）＋「戻す」（自動配分値へ復帰）＋ピン。
- 頂点を間引く方式で **UV・法線・ボーンウェイト・ブレンドシェイプ（表情）を保持** します。**顔・髪は品質下限で強く保護**。Unity Cloth の付いたメッシュは自動で対象外です。大きく削る場合は先に外部ツール（Blender等）で整えるのが推奨です。

### ⑥ マテリアル（PC / Quest）

PC列（アトラス統合）とQuest列（変換方法）を並べて設定します。

**PC（アトラス統合）**
- 「PCマテリアルをアトラス統合」トグル。互換マテリアルを1枚のアトラスへまとめてスロット数・テクスチャメモリを削減します。
- オプション: アトラス最大サイズ（1024 / 2048）、**テクスチャ無し（色だけ）の材質も統合**（色をセルへ焼き込む）、**エミッションをアトラスへ焼き込む**、**カリング差を無視**（統合後は両面描画）、**アウトライン**（`統合しない / 外して統合（推奨） / 付きに統一`）。
- 統合予定グループ（両面化・アウトライン付与/除去の注記つき）と、統合できない/単独マテリアルを **理由つき** で表示。マテリアルごとにアトラス除外も指定できます。

**Quest / iOS（マテリアル変換）**
- **変換先シェーダー**: `Toon Standard`（影ランプ等対応） / `Toon Lit`（最軽量）。
- **半透明の既定処理（透過3モード）**: `自動で半透明を再現（Emulate・推奨） / 非表示にする（Hide） / 不透明に変換（Opaque）`。
- 各マテリアルの **個別変換方法（7種）**: `自動 / Toon Standard / Toon Lit / 加算（半透明） / 乗算（半透明） / 非表示 / 変換しない`。行ごとに「変換後にどうなるか」を表示。
- **状態バッジ**: 透過 / カットアウト / パーティクル / アニメ使用 / メニュー・ギミック参照 / TMP / 破損 / 対応済。
- Quest版で非表示になる予定のマテリアルがあると件数を案内し、その **非表示行にクイックボタン「不透明にする」「乗算で再現」** を出します（ストッキング等を見えるように戻せます）。

**Quest固有：アトラス統合（実験的）**
- 「アトラス統合を有効化」＋アトラス最大サイズ（1024 / 2048 / 4096）＋「**影ランプを統一してグループ化**」（影ランプの違いを無視してスロット削減優先。影ランプが個別生成されるアバターでオフだと統合がほぼ効かない）。マテリアルごとに統合対象へ含める/外すを指定でき、**統合できない理由の見方**（アニメ差し替え・UVタイリング等）も対象外行に表示します。

**Quest固有：表情デカール（チーク / 涙 / アイハイライト）**
- 既定処理が Emulate のときは乗算/加算で再現され非表示化されない旨を案内。Hide / Opaque のときは非表示化される旨と、検出されたデカール一覧（ピンつき）を表示します。

**Quest / iOS 変換設定（実行前の最終確認・末尾）**
- 最大テクスチャサイズ（512 / 1024推奨 / 2048）、Android圧縮形式（ASTC 4x4 / 5x5 / 6x6推奨 / 8x8）。
- シェーダー別: **影ランプを生成**（Toon Standard）、**エミッションを変換**、**影をメインテクスチャへベイク**（Toon Lit）、**リムライトを近似変換**（Toon Standard・既定オフ）。
- **単色・低ディテールなテクスチャを極限まで縮小**（既定オン）。
- 詳細設定: アニメーションも変換 / ⚠非対応コンポーネントを削除 / UnityコンストレイントをVRCConstraintへ変換 / ⚠マージ後もPoor上限超過分を削除 / 出力先フォルダ（既定に戻す）/ **Android・iOSビルド時に非対応コンポーネントを自動削除（Auto Fix相当）**。

### ⑦ 実行

- **取り残された複製の再確認**: 元アバターをリネームした後に残った古い『_Opt』『_Quest』複製があると名前付きで警告します（自動削除はしません）。
- **実行オプション**: 「生成した複製をプレファブとして保存する」「AAO（Trace and Optimize）を複製へ付与する」。
- **この実行で行われること**: 対象・目標ランク・透過処理・ポリゴン削減の有無などのプリフライト要約を箇条書き表示。Quest対象なのにビルドターゲットがAndroidでない場合は注意を出します。
- **実行（選択した対象へ生成）**: 確認ダイアログの後、選択対象へ複製を生成します。元アバターは無改変。
- **実行結果**: パイプラインごとのレポート（コピー可）、**変換前後の比較テーブル**（項目 / 変換前 / 変換後 / 判定。PC・Quest列で目標と突き合わせ）、**変換結果チェック**（元 ← → 変換後マテリアルのサムネイル比較）。
- **アップロード手順ガイド（4手順）** が常設されます。
  1. ビルドターゲットを **Windows** に切替 → SDKパネルで **_Opt** をアップロード。
  2. PC版の **Blueprint ID を Quest版へ同期**（別IDがある場合は確認ダイアログ）。
  3. ビルドターゲットを **Android** に切替。
  4. SDKパネルで **_Quest** をアップロード。PC版と同じBlueprint IDのまま上げると、1つのアバターのPC / Quest両対応（フォールバック）になります。
  現在のビルドターゲットと各複製のBlueprint IDを状態表示し、_Optと_Questが同一IDかどうかも色付きで示します。

**繰り返し再生成の安全性**: 元プレファブは非破壊で、生成物は別の複製です。実行のたびに前回の『_Opt』『_Quest』複製は自動で作り直されるため複製は蓄積しません。生成アセットは決まった場所へGUID保持で上書きされ、参照は壊れません（Quest変換ではアニメーターコントローラーのみ技術的制約で作り直しになります）。

**MA（Modular Avatar）互換**: 変換はビルド前のクローン編集で行うため、MAがビルド時に統合・削除・パス乗っ取りする対象と齟齬が起こり得ます。本ツールはMA参照をすべてリフレクション経由で監査し（未導入時は自動でno-op）、除外/非表示サブツリーへのMA参照、Merge Armature使用時の計測注記、ShapeChangerの二重削除回避、Merge Animator経由クリップの識別、理解外のMAコンポーネント検出などを **自動でガード・警告** します。

## 5. 軽くなる仕組み

| 処理 | 何が減るか | 備考 |
|---|---|---|
| **トグル固定（常時表示）** | SkinnedMesh数 | 表示アニメを常時ONに固定 → AAOがビルド時に同一メッシュへ結合（**要AAO**） |
| **トグル固定（非表示に固定）** | SkinnedMesh数 | 使わない衣装をEditorOnly化して除去（AAO不要） |
| **SkinnedMesh統合** | SkinnedMesh数 | 顔（ビセーム/まばたき）以外を1つへ（結合はビルド時にAAO） |
| **アトラス統合** | マテリアルスロット数 | 複数マテリアルのテクスチャを1枚へ（同一メッシュ内は変換時、メッシュをまたぐ統合はビルド時にAAO） |
| **テクスチャ縮小** | テクスチャメモリ（VRAM）・DLサイズ | 解像度を下げる |
| **ポリゴン削減** | 三角数 | 頂点を間引く（表情のブレンドシェイプ・UVは保持） |
| **PhysBone整理** | PhysBoneコンポーネント数 | 揺れもの設定を選別・マージ |
| **透過処理** | （ランク削減ではない） | Questに半透明シェーダーが無いため、乗算/加算のParticlesシェーダーで見た目を近似（Quest対応のための処理） |

> **ビルド時適用の注意**: AAO（Trace and Optimize）によるメッシュ / スロット統合や隠れメッシュ削減は **ビルド（アップロード）時に反映** されます。そのため生成直後にシーン上・診断表で見える数値は最終的なビルド後より控えめ（改善が少なめ）に出ます。実際のアップロード後はさらに軽くなる場合があります。最終確認はSDKのビルド結果で行ってください。

## 6. トラブルシューティング

**「見えない / 消えた」系**
- **Quest版で髪・衣装が消えた**: 大型メッシュ・髪は透過自動非表示の対象外です。再変換すれば直ります。透過衣類（ストッキング等）が消えた場合は、⑥マテリアルの該当行で「不透明にする」「乗算で再現」を選びます。
- **メニューで切り替える目・衣装がQuestで消える**: メニュー / ギミック参照のマテリアルも自動変換されます。⑥で「メニュー・ギミック参照」バッジが付いているか確認してください。
- **瞳のテクスチャが消える（PC）**: MAのMaterial Setter等で差し替わる瞳マテリアルは自動でアトラス除外されます。残る場合は手動でアトラス除外に指定します。

**「見える / 出てしまう」系**
- **チーク等が顔に板として乗る**: 既定は「自動で半透明を再現」で乗算/加算に近似されます。板に見える場合は⑥で既定処理や個別指定を確認します。
- **変換後に一部が真っ白に光る（白髪化含む）**: エミッション引き継ぎが原因のことが多く、最新版では自動対処されます。再変換で改善。残る場合は生成マテリアルでEmissionを手動調整します。
- **まぶた等に謎のハイライト**: 「リムライトを近似変換」をオフのままにしてください（既定オフ）。

**サイズ・ランク系**
- **ダウンロードサイズが10MBを超える**: 効果が大きい順に、①全テクスチャを1024px（さらに顔・体以外を512px）、②ASTCブロックサイズを大きく（6x6→8x8で約半分）、③使わないオブジェクトをQuest除外、④AAO Trace and Optimizeを有効化、⑤メッシュ側の削減（ポリゴン削減・不要ブレンドシェイプ削除）。
- **Very Poorになる**: 素体+衣装はスキンメッシュ数が上限を超えやすいのが主因です。SkinnedMesh統合・トグル固定でメッシュ数を減らし、AAOを導入してください。スロットが減らないときは、SkinnedMesh数そのものを減らす（SkinnedMesh統合）とスロットも同時に下がります。

**古い複製 / 変換が効かない系**
- **変換したのに軽くならない / PhysBoneが減っていない**: 前回の古い複製をアップロードしている可能性が高いです。最新版は再変換時に古い複製を自動削除して新しい複製をハイライトするので、ハイライトされた複製をアップロードしてください（SDKパネルの「Previous build of ...」は前回ビルドの数値です）。
- **SDKビルド画面に赤いシェーダーアラート**: 元アバターを選んでいないか確認。アップロードするのは『_Opt』『_Quest』の方です。
- **生成マテリアルのシェーダーを手で変えても戻る**: 生成された複製（『_Opt』『_Quest』）のシェーダーを手動で差し替えても、次の再変換（再生成）で上書きされます。恒久的に変えたい場合は⑥マテリアルの該当行「変換方法」で指定してください。

> いずれの診断値も **エディタ上の状態で計算** されるため、AAO等がビルド時に統合・削減する分は反映されず実際より悪めに出ることがあります。サイズも推定値です。最終確認はSDKのビルド結果で行ってください。
